## 一、任务规则与测评体系

### 1.1 角色分流机制

全班小组通过抽签分为两类角色：
- **迷宫设计组**：生成具有挑战性的完美迷宫
- **AI 玩家组**：设计能够在局部视野下高效穿越迷宫的智能体

### 1.2 核心测评指标

AI 玩家的核心考核指标定义为**资源效率比**（Resource Efficiency Ratio）：

$$
\text{Score}_{\text{AI}} = \frac{R_{\text{remaining}}}{S_{\text{total}}}
$$

其中：
- $R_{\text{remaining}}$：抵达终点时剩余的资源价值（金币价值 50，陷阱价值 -30）
- $S_{\text{total}}$：从起点到终点所用的总步数

该指标的设计意图：**鼓励 AI 在尽可能少的步数内收集尽可能多的净资源**。

### 1.3 交叉测试矩阵

交叉测试采用**全矩阵评估**（Full Matrix Evaluation）机制：

| 测试维度 | 迷宫组得分依据 | AI 玩家组得分依据 |
|:---:|:---:|:---:|
| 单组对单组 | 全班 AI 在该迷宫上的失败率、平均得分 | 在该迷宫上的 $\text{Score}_{\text{AI}}$ |
| 最终得分 | 全班 AI 评估的综合得分 | 全部迷宫评估的平均得分 |

**关键洞察**：迷宫组的得分与 AI 玩家的平均表现成反比，而 AI 玩家的得分与迷宫的平均难度成反比。这构成了一个**零和博弈**（Zero-Sum Game）结构。

## 二、博弈论视角：迷宫设计与 AI 玩家的对抗关系

### 2.1 对抗性程序内容生成（Adversarial PCG）

本次课设的核心机制与学术界的**对抗性程序内容生成**（Adversarial Procedural Content Generation）高度吻合。在该框架下：

- **迷宫设计**对应**环境生成器**（Environment Generator）：负责生成具有特定难度分布的迷宫拓扑与资源布局
- **AI 玩家**对应**策略优化器**（Policy Optimizer）：在受限信息流（3×3 视野）下逼近全局最优决策

与传统 GAN 不同的是，本课设的对抗关系是**解耦的**：迷宫设计者无法直接获取 AI 的梯度信息，只能通过评估 AI 的表现来迭代优化迷宫。

### 2.2 协同进化（Coevolution）的工程意义

两类任务在工程层面存在**双向依赖**：

1. **迷宫组的依赖**：要设计出"最具挑战性"的迷宫，必须在组内实现一个 AI 玩家来测试迷宫的防御能力
2. **AI 组的依赖**：要训练出"最具泛化性"的 AI，必须在组内生成大量极端迷宫作为训练样本

这种双向依赖在学术界被称为**协同进化**（Coevolution），其核心思想是：两个种群（迷宫种群与 AI 策略种群）在对抗中共同进化（Rosin & Belew, 1997）。

## 三、方案甲：迷宫设计任务

### 3.1 四种迷宫生成算法对比

任务书要求使用分治、贪心（最小生成树）、回溯（DFS）、分支限界（BFS）四种范式生成完美迷宫。以下从算法特性角度进行对比分析：

| 算法范式 | 典型实现 | 时间复杂度 | 空间复杂度 | 生成迷宫特性 |
|:---:|:---:|:---:|:---:|:---:|
| 分治法 | 递归分割 | $O(n^2)$ | $O(\log n)$（递归栈） | 区块结构清晰，走廊较长 |
| 贪心法 | 随机 Kruskal / Prim | $O(n^2 \log n)$ | $O(n^2)$ | 分支均匀，路径多样 |
| 回溯法 | 随机 DFS | $O(n^2)$ | $O(n^2)$（显式栈） | 长走廊多，死胡同少 |
| 分支限界 | BFS 生长 | $O(n^2)$ | $O(n^2)$ | 路径最短，分支密集 |

> **注**：$n$ 为迷宫边长，完美迷宫的数学定义为：连通、无环、任意两点间存在唯一路径。

**伪代码：随机 DFS 迷宫生成**

```
function GenerateMaze_DFS(n):
    maze ← n × n 网格，初始全为墙壁
    start ← 随机选择一个奇数坐标
    stack ← [start]
    maze[start] ← 通路

    while stack 非空:
        current ← stack.top()
        neighbors ← 获取 current 的未访问邻居（距离为 2）
        if neighbors 非空:
            next ← 随机选择 neighbors
            打通 current 与 next 之间的墙壁
            maze[next] ← 通路
            stack.push(next)
        else:
            stack.pop()

    return maze
```

### 3.2 基于遗憾值的迷宫优化（PAIRED 思想落地）

Dennis et al. (2020) 提出的 **PAIRED**（Protagonist Antagonist Induced Regret Environment Design）框架，其核心思想是通过最大化**遗憾值**（Regret）来生成具有挑战性的环境。

**PAIRED 三玩家博弈机制**：

1. **Adversary（环境生成器）**：生成迷宫拓扑和资源分布
2. **Protagonist（主角智能体）**：能力受限的 AI（如 3×3 贪心策略）
3. **Antagonist（对抗者智能体）**：能力更强的 AI（如拥有全局视野）

**遗憾值定义**：

$$
\text{Regret} = V_{\text{Antagonist}} - V_{\text{Protagonist}}
$$

生成器的目标是最大化遗憾值。这意味着：
- 迷宫必须是**可解的**（否则 Antagonist 得分为 0，遗憾值为 0）
- 迷宫必须对 Protagonist **极度不友好**

**课设落地映射**：

| PAIRED 角色 | 课设对应 | 说明 |
|:---:|:---:|:---:|
| Adversary | 迷宫生成算法 | 使用四种范式生成候选迷宫 |
| Protagonist | 3×3 贪心策略 | 模拟全班 AI 的平均水平 |
| Antagonist | 动态规划最优解 | 任务书要求的 DP 算法 |

**迷宫优化目标**：

$$
\text{Fitness}_{\text{Maze}} = V_{\text{DP}} - V_{\text{Greedy}}
$$

其中 $V_{\text{DP}}$ 为 DP 算出的理论最高得分，$V_{\text{Greedy}}$ 为 3×3 贪心策略的实际得分。

### 3.3 动态规划资源路径与 BOSS 战参数设计

#### 3.3.1 最优资源收集路径（DP）

在迷宫矩阵中，金币和陷阱的分布构成一个**带权网格图**。最优资源收集路径问题可建模为：从起点出发，在到达终点的约束下，最大化收集的净资源价值。

**状态定义**：

$$
dp[i][j] = \text{从起点到达位置}(i,j)\text{所能获得的最大净资源价值}
$$

**状态转移方程**：

$$
dp[i][j] = \max(dp[i-1][j],\ dp[i][j-1],\ dp[i+1][j],\ dp[i][j+1]) + \text{value}(i,j)
$$

其中 $\text{value}(i,j)$ 为位置 $(i,j)$ 的资源价值（金币 50，陷阱 -30，空地 0）。

**伪代码：资源收集 DP**

```
function OptimalResourcePath(maze, start, end):
    n ← maze.size
    dp ← n × n 数组，初始化为 -∞
    parent ← n × n 数组，记录最优路径来源
    dp[start] ← value(start)

    queue ← [start]  // BFS 遍历顺序
    while queue 非空:
        (i, j) ← queue.dequeue()
        for each neighbor (ni, nj) of (i, j):
            if maze[ni][nj] ≠ 墙壁 and dp[i][j] + value(ni, nj) > dp[ni][nj]:
                dp[ni][nj] ← dp[i][j] + value(ni, nj)
                parent[ni][nj] ← (i, j)
                queue.enqueue((ni, nj))

    // 回溯最优路径
    path ← 从 end 回溯 parent 数组至 start
    return dp[end], path
```

#### 3.3.2 BOSS 战策略优化（分支限界法）

BOSS 战的核心问题是：**在限定回合数内，使用最少的金币击败 BOSS**。

**问题建模**：

- 输入：BOSS 血量 $H$，AI 技能集合 $\{s_1, s_2, \ldots, s_k\}$，每个技能有伤害 $d_i$ 和冷却回合 $c_i$
- 约束：至少有一个技能无冷却（如普通攻击）
- 输出：最少回合数、最优技能序列、复活所需金币数

**分支限界法求解**：

将技能序列的搜索空间组织为一棵状态树，每个节点表示一个回合的状态（BOSS 剩余血量、各技能冷却状态）。使用**上界函数**（BOSS 剩余血量 / 最大单回合伤害）进行剪枝。

**伪代码：分支限界 BOSS 战**

```
function BranchAndBound_BOSS(H, skills):
    best_rounds ← ∞
    best_sequence ← null

    // 优先队列，按回合数排序
    pq ← PriorityQueue()
    pq.enqueue((H, [], 各技能冷却状态))

    while pq 非空:
        (remaining_hp, sequence, cooldowns) ← pq.dequeue()

        if remaining_hp ≤ 0:
            if len(sequence) < best_rounds:
                best_rounds ← len(sequence)
                best_sequence ← sequence
            continue

        // 上界剪枝
        upper_bound ← len(sequence) + ⌈remaining_hp / max_damage⌉
        if upper_bound ≥ best_rounds:
            continue

        for each skill s in skills:
            if cooldowns[s] = 0:  // 技能可用
                new_hp ← remaining_hp - s.damage
                new_cooldowns ← 更新冷却状态
                pq.enqueue((new_hp, sequence + [s], new_cooldowns))

    return best_rounds, best_sequence
```

**BOSS 战参数设计策略**：

利用分支限界法计算出击败 BOSS 的**最少金币消耗** $M$，然后通过 DP 算法将迷宫的理论最高可收集金币数控制在 $M + \epsilon$（$\epsilon$ 为极小正数）。

**效果**：AI 玩家在迷宫探索阶段只要因 3×3 盲区稍微绕路或错过 1 个金币，到达 BOSS 关卡时就会因金币不足而 GAME OVER。

## 四、方案乙：AI 玩家设计任务

### 4.1 环境建模与 Gymnasium 接口设计

AI 玩家只有 3×3 的局部视野，无法感知全局状态。这在学术界被称为**部分可观测马尔可夫决策过程**（Partially Observable Markov Decision Process, POMDP）。

**标准环境接口定义**（参考 `mazelab` 框架设计）：

```
class MazeEnv:
    属性:
        maze: n × n 矩阵（0=墙壁, 1=通路, 2=金币, 3=陷阱, 4=BOSS）
        player_pos: (row, col) 当前位置
        player_gold: int 当前金币数
        player_hp: int 当前血量

    方法:
        reset() → observation
            重置环境，返回初始 3×3 视野

        step(action) → (observation, reward, done, info)
            action ∈ {上, 下, 左, 右}
            observation: 3×3 局部视野矩阵
            reward: 即时奖励（金币 +50, 陷阱 -30, 步数惩罚 -1）
            done: 是否到达终点或 GAME OVER
            info: {gold, hp, steps, boss_defeated}

        get_3x3_view() → 3 × 3 numpy array
            返回以玩家为中心的 3×3 区域
```

### 4.2 基线方案：3×3 贪心策略

**算法描述**：每次移动时，在 3×3 视野内选择"性价比"最高的资源。

**性价比定义**：

$$
\text{Value}(r) = \frac{\text{resource\_value}(r)}{\text{distance}(player, r)}
$$

**伪代码：贪心资源拾取**

```
function GreedyPolicy(observation, player_pos):
    candidates ← 在 3×3 视野内识别所有可见资源
    if candidates 为空:
        return 随机选择一个可行方向

    best_resource ← argmax(Value(r)) for r in candidates
    direction ← 计算从 player_pos 到 best_resource 的移动方向

    if direction 方向为墙壁:
        return 随机选择一个可行方向

    return direction
```

**局限性**：贪心策略容易被"伪梯度"误导——在 T 字路口边缘放置高价值金币诱导 AI 进入死胡同。

### 4.3 进阶方案：DRQN 记忆增强

Hausknecht & Stone (2015) 提出的 **DRQN**（Deep Recurrent Q-Network）通过在 DQN 中引入 LSTM 层，使智能体能够记忆历史轨迹，适用于 POMDP 环境。

**网络架构**：

$$
\text{Input}(3 \times 3) \rightarrow \text{Conv2D} \rightarrow \text{Flatten} \rightarrow \text{LSTM} \rightarrow \text{FC} \rightarrow Q(a)
$$

**状态空间设计**：

| 输入维度 | 内容 | 说明 |
|:---:|:---:|:---:|
| 3×3×C | 局部视野 | C 为通道数（墙壁/通路/金币/陷阱） |
| 1 | 当前金币数 | 归一化到 [0, 1] |
| 1 | 当前血量 | 归一化到 [0, 1] |
| H | LSTM 隐状态 | 记忆历史轨迹信息 |

**奖励函数设计**：

$$
R_t = \beta_1 \cdot \Delta \text{Gold} - \beta_2 \cdot \Delta \text{Trap} - \beta_3 \cdot \text{Step\_Penalty} + \beta_4 \cdot \text{Terminal\_Bonus}
$$

建议参数：$\beta_1 = 1.0, \beta_2 = 0.6, \beta_3 = 0.01, \beta_4 = 10.0$

通过增大 $\beta_3$（步数惩罚），迫使智能体学会为了高资源效率比而放弃无效探索。

### 4.4 课程学习与泛化性训练

Cobbe et al. (2020) 在 **Procgen Benchmark** 研究中指出，利用程序化生成的无限多样性环境进行训练，可以迫使 RL 网络提取出不依赖特定环境拓扑的"高阶特征"。

**三阶段训练流水线**：

```
阶段一：常规随机迷宫
    目标：学会基础行为（吃金币、避墙、到达终点）
    训练量：~1000 个随机迷宫

          ↓

阶段二：对抗性迷宫
    目标：学会高级决策（抵御诱导陷阱、权衡绕路与风险）
    训练量：~500 个极端迷宫（含伪梯度、连环陷阱等）

          ↓

阶段三：跨组验证
    目标：泛化性检验，超参微调
    数据源：其他组提供的迷宫样本（如有）
```

### 4.5 BOSS 战策略解耦

BOSS 战是离散回合制且有明确的技能/复活机制，其最优策略可以通过**精确算法**（分支限界/DP）求解，无需使用 RL。

**解耦设计**：
- **迷宫探索阶段**：使用 DRQN 最大化带出金币
- **BOSS 战阶段**：切换为分支限界法的状态机，以理论最高胜率击败 BOSS

这种解耦设计的优势：将连续决策问题（迷宫探索）与离散优化问题（BOSS 战）分开处理，各自使用最适合的算法范式。

## 五、工程开发流水线

### 5.1 基础设施先行

无论抽到哪类任务，第一步都是封装标准的迷宫环境接口。

**推荐技术栈**：
- Python 3.8+
- NumPy（迷宫矩阵操作）
- Gymnasium（环境接口标准）
- PyTorch（AI 玩家的深度学习框架，如需要）
- Matplotlib/Pygame（可视化）

**环境封装要点**：
1. `step()` 函数必须返回 3×3 视野（而非全局地图）
2. 奖励函数需同时考虑资源收集和步数消耗
3. 支持迷宫矩阵的导入/导出（用于交叉测试）

### 5.2 方案甲开发路线（迷宫设计组）

```
第一步：实现四种迷宫生成算法
    ├── 分治法（递归分割）
    ├── 贪心法（随机 Kruskal/Prim）
    ├── 回溯法（随机 DFS）
    └── 分支限界（BFS 生长）

第二步：实现动态规划最优路径算法
    └── 输出：理论最高得分 V_DP 和最优路径

第三步：实现组内测试 AI
    └── 3×3 贪心策略，用于计算 V_Greedy

第四步：实现 BOSS 战参数优化
    └── 分支限界法计算最少回合数和金币门槛

第五步（可选）：实现遗传算法自动加固器
    └── 以 (V_DP - V_Greedy) 为适应度，迭代优化迷宫
```

### 5.3 方案乙开发路线（AI 玩家组）

```
第一步：实现标准迷宫环境
    └── 遵循 Gymnasium 接口规范

第二步：实现基线策略
    └── 3×3 贪心策略，用于性能对比

第三步：实现进阶策略（可选）
    ├── 方案 A：Q-Learning（表格型，适合小迷宫）
    ├── 方案 B：DRQN（深度循环网络，适合大迷宫）
    └── 方案 C：PPO + LSTM（策略梯度方法）

第四步：实现 BOSS 战策略
    └── 分支限界法或 DP，解耦处理

第五步：课程学习训练流水线
    └── 三阶段训练，提升泛化性
```

### 6.3 相关研究方向

- **非监督环境设计**（Unsupervised Environment Design, UED）：自动生成具有特定难度分布的训练环境
- **基于 RL 的程序内容生成**（PCGRL）：使用强化学习训练关卡设计智能体
- **协同进化算法**（Coevolutionary Algorithms）：两个种群在对抗中共同进化
