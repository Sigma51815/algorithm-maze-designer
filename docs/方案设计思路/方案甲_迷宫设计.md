# 方案甲：迷宫设计任务

> **目标**：生成具有挑战性的完美迷宫，使全班 AI 玩家的平均得分最低  
> **核心算法**：分治、贪心（MST）、回溯（DFS）、分支限界（BFS）  
> **进阶策略**：基于遗憾值的迷宫优化（PAIRED 思想）

---

## 一、任务分析

### 1.1 评分机制

迷宫组的得分取决于全班 AI 玩家在该迷宫上的表现：

$$
\text{Score}_{\text{Maze}} = f(\text{AI 失败率},\ \text{AI 平均得分},\ \text{AI 破产率})
$$

**设计目标**：在保证迷宫可解的前提下，最大化 AI 的失败率和破产率，最小化 AI 的平均得分。

### 1.2 约束条件

| 约束 | 要求 |
|:---:|:---:|
| 连通性 | 完美迷宫（无环、连通、任意两点唯一路径） |
| 尺寸 | $n \times n$，建议 $n \geq 15$ |
| 资源分布 | 金币（价值 50）、陷阱（价值 -30） |
| BOSS 战 | 需击败守卫终点的 BOSS |

---

## 二、四种迷宫生成算法

### 2.1 算法对比

| 算法 | 范式 | 时间复杂度 | 空间复杂度 | 迷宫特性 |
|:---:|:---:|:---:|:---:|:---:|
| 递归分割 | 分治 | $O(n^2)$ | $O(\log n)$ | 区块结构清晰，走廊较长 |
| 随机 Kruskal | 贪心 | $O(n^2 \log n)$ | $O(n^2)$ | 分支均匀，路径多样 |
| 随机 DFS | 回溯 | $O(n^2)$ | $O(n^2)$ | 长走廊多，死胡同少 |
| BFS 生长 | 分支限界 | $O(n^2)$ | $O(n^2)$ | 路径最短，分支密集 |

### 2.2 伪代码实现

#### 算法 1：递归分割（分治法）

```
function RecursiveDivision(maze, x, y, width, height):
    if width < 3 or height < 3:
        return

    // 选择分割方向（优先分割较长的边）
    horizontal ← height > width ? true : (width > height ? false : random())

    if horizontal:
        // 水平分割：在中间行放置墙壁，留一个通道
        wall_y ← y + random_even(1, height - 1)
        passage_x ← x + random_odd(0, width - 1)
        for i = x to x + width - 1:
            if i ≠ passage_x:
                maze[wall_y][i] ← 墙壁
        // 递归处理上下两个子区域
        RecursiveDivision(maze, x, y, width, wall_y - y)
        RecursiveDivision(maze, x, wall_y + 1, width, y + height - wall_y - 1)
    else:
        // 垂直分割：在中间列放置墙壁，留一个通道
        wall_x ← x + random_even(1, width - 1)
        passage_y ← y + random_odd(0, height - 1)
        for j = y to y + height - 1:
            if j ≠ passage_y:
                maze[j][wall_x] ← 墙壁
        // 递归处理左右两个子区域
        RecursiveDivision(maze, x, y, wall_x - x, height)
        RecursiveDivision(maze, wall_x + 1, y, x + width - wall_x - 1, height)
```

#### 算法 2：随机 Kruskal（贪心法）

```
function RandomKruskal(n):
    maze ← n × n 网格，初始全为墙壁
    uf ← UnionFind(n²)  // 并查集

    // 收集所有可能的内墙（连接两个奇数坐标的墙）
    walls ← []
    for i = 1 to n-1 step 2:
        for j = 1 to n-1 step 2:
            if i + 2 < n: walls.append((i, j, i+2, j))  // 水平边
            if j + 2 < n: walls.append((i, j, i, j+2))  // 垂直边

    shuffle(walls)  // 随机打乱

    for (x1, y1, x2, y2) in walls:
        id1 ← y1 * n + x1
        id2 ← y2 * n + x2
        if not uf.connected(id1, id2):
            uf.union(id1, id2)
            // 打通墙壁
            maze[(y1 + y2) / 2][(x1 + x2) / 2] ← 通路
            maze[y1][x1] ← 通路
            maze[y2][x2] ← 通路

    return maze
```

#### 算法 3：随机 DFS（回溯法）

```
function RandomDFS(n):
    maze ← n × n 网格，初始全为墙壁
    start ← (1, 1)
    stack ← [start]
    maze[start.y][start.x] ← 通路

    while stack 非空:
        current ← stack.top()
        neighbors ← 获取 current 的未访问邻居（距离为 2）

        if neighbors 非空:
            next ← 随机选择 neighbors
            // 打通 current 与 next 之间的墙壁
            maze[(current.y + next.y) / 2][(current.x + next.x) / 2] ← 通路
            maze[next.y][next.x] ← 通路
            stack.push(next)
        else:
            stack.pop()

    return maze
```

#### 算法 4：BFS 生长（分支限界法）

```
function BFSGrowth(n):
    maze ← n × n 网格，初始全为墙壁
    start ← (1, 1)
    maze[start.y][start.x] ← 通路
    frontier ← [start]  // 前沿集合

    while frontier 非空:
        current ← 随机选择 frontier 中的一个
        neighbors ← 获取 current 的未访问邻居（距离为 2）

        if neighbors 非空:
            next ← 随机选择 neighbors
            maze[(current.y + next.y) / 2][(current.x + next.x) / 2] ← 通路
            maze[next.y][next.x] ← 通路
            frontier.append(next)

        if 无未访问邻居:
            frontier.remove(current)

    return maze
```

## 三、最优资源收集路径（动态规划）

### 3.1 问题建模

在迷宫中，金币和陷阱构成带权网格图。目标：从起点出发，到达终点，最大化净资源价值。

### 3.2 算法设计

**状态定义**：

$$
dp[i][j] = \text{从起点到达位置}(i,j)\text{所能获得的最大净资源价值}
$$

**状态转移**：

$$
dp[i][j] = \max \begin{cases} dp[i-1][j] \\ dp[i][j-1] \\ dp[i+1][j] \\ dp[i][j+1] \end{cases} + \text{value}(i,j)
$$

**伪代码**：

```
function OptimalResourcePath(maze, start, end):
    n ← maze.size
    dp ← n × n 数组，初始化为 -∞
    parent ← n × n 数组
    dp[start] ← value(start)

    // BFS 遍历（需保证无环路依赖）
    queue ← 按拓扑序遍历所有可达位置
    for each (i, j) in queue:
        for each neighbor (ni, nj) of (i, j):
            if maze[ni][nj] ≠ 墙壁:
                if dp[i][j] + value(ni, nj) > dp[ni][nj]:
                    dp[ni][nj] ← dp[i][j] + value(ni, nj)
                    parent[ni][nj] ← (i, j)

    path ← 从 end 回溯 parent 至 start
    return dp[end], path
```

### 3.3 复杂度分析

- **时间复杂度**：$O(n^2)$（每个位置访问一次）
- **空间复杂度**：$O(n^2)$（dp 数组 + parent 数组）

## 四、BOSS 战参数设计（分支限界法）

### 4.1 问题建模

- 输入：BOSS 血量 $H$，技能集合 $\{s_1, \ldots, s_k\}$
- 约束：技能有冷却时间，至少一个技能无冷却
- 输出：最少回合数、最优技能序列、复活金币数

### 4.2 分支限界算法

```
function BranchAndBound_BOSS(H, skills):
    best_rounds ← ∞
    best_seq ← null
    pq ← PriorityQueue()  // 按回合数排序

    pq.enqueue((H, [], {s.cooldown = 0 for s in skills}))

    while pq 非空:
        (hp, seq, cd) ← pq.dequeue()

        if hp ≤ 0:
            if len(seq) < best_rounds:
                best_rounds ← len(seq)
                best_seq ← seq
            continue

        // 上界剪枝
        ub ← len(seq) + ⌈hp / max(s.damage for s in skills)⌉
        if ub ≥ best_rounds:
            continue

        for each skill s in skills:
            if cd[s] = 0:
                new_hp ← hp - s.damage
                new_cd ← update_cooldowns(cd, s)
                pq.enqueue((new_hp, seq + [s], new_cd))

    return best_rounds, best_seq
```

### 4.3 边界压榨策略

利用 DP 计算迷宫理论最高可收集金币数 $V_{\text{max}}$，然后设置 BOSS 战参数使得：

$$
M_{\text{required}} = V_{\text{max}} - \epsilon
$$

其中 $M_{\text{required}}$ 为击败 BOSS 所需的最少金币，$\epsilon$ 为极小正数。

**效果**：AI 只要在迷宫探索阶段因 3×3 盲区错过 1 个金币，就会因金币不足而 GAME OVER。

## 五、进阶：基于遗憾值的迷宫优化

### 5.1 PAIRED 思想落地

Dennis et al. (2020) 提出的 PAIRED 框架核心思想：

$$
\text{Regret} = V_{\text{Antagonist}} - V_{\text{Protagonist}}
$$

**课设映射**：

| PAIRED 角色 | 课设对应 |
|:---:|:---:|
| Adversary | 迷宫生成算法 |
| Protagonist | 3×3 贪心策略（模拟全班 AI） |
| Antagonist | DP 最动规划（理论最优） |

**迷宫适应度**：

$$
\text{Fitness}_{\text{Maze}} = V_{\text{DP}} - V_{\text{Greedy}}
$$

### 5.2 遗传算法加固器（可选）

```
function GeneticMazeOptimizer(population_size, generations):
    // 初始化种群：随机生成迷宫
    population ← [RandomMaze() for _ in range(population_size)]

    for gen = 1 to generations:
        // 评估适应度
        fitness ← []
        for maze in population:
            v_dp ← OptimalResourcePath(maze).value
            v_greedy ← GreedyAgent(maze).value
            fitness.append(v_dp - v_greedy)

        // 选择：保留高适应度个体
        elite ← select_top_k(population, fitness, k = population_size / 2)

        // 交叉与变异
        offspring ← []
        while len(offspring) < population_size / 2:
            parent1, parent2 ← random_pair(elite)
            child ← crossover(parent1, parent2)
            child ← mutate(child, mutation_rate = 0.1)
            offspring.append(child)

        population ← elite + offspring

    return argmax(population, fitness)
```
