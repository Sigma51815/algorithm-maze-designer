# 迷宫优化器 — 训练思路与指标说明

> 生成时间：2026-06-15

---

## 一、优化思路

### 总体方案：GA + RL 两阶段优化

1. **遗传算法（GA）**：生成多样化迷宫种群，通过选择、交叉、变异迭代进化，搜索最优迷宫设计
2. **RL 精调（可选）**：对 GA 每代最优解用 Q-Learning 微调资源布局，跳出局部最优

### GA 流程

```
初始种群（随机生成迷宫）
    ↓
适应度评估（DP / 贪心 / 拓扑）
    ↓
选择（锦标赛）→ 交叉 → 变异
    ↓
新一代种群
    ↓ （循环 N 代）
输出最优迷宫
```

### RL 精调流程

在 GA 每一代结束后，对最优个体进行 Q-Learning 训练：
- **状态**：迷宫中金币/陷阱的当前布局
- **动作**：移动某个资源到相邻空位
- **奖励**：布局变化后的适应度增益
- 经过 `rl-episodes` 轮训练后，返回优化后的布局

## 二、输出指标说明

运行时每代输出如下格式：

```
Gen   1/30 | best=547.5 avg=415.0 | dp=380 greedy=-40 regret=420
```

| 指标 | 全称 | 含义 |
|------|------|------|
| **best** | Best Fitness | 当前代中最优迷宫的综合适应度分数，越高越好 |
| **avg** | Average Fitness | 当前代所有个体的平均适应度，反映种群整体水平 |
| **dp** | DP Optimal Score | 动态规划玩家（全局最优策略）走迷宫能获得的最高金币价值 |
| **greedy** | Greedy AI Score | 3×3 局部贪心 AI 走迷宫的得分，通常为负值（贪心会踩陷阱） |
| **regret** | Regret = dp - greedy | 遗憾值，衡量迷宫对"聪明玩家 vs 鲁莽玩家"的区分度 |

### 最终输出

```
=== Result ===
DP optimal: 380        # DP 最优得分
Greedy worst: -40      # 贪心最差得分
Regret: 420            # 遗憾值
Topo difficulty: 1.16  # 拓扑难度（路径复杂度）
Final fitness: 547.5   # 最终综合适应度
Valid: YES             # 迷宫是否合法（连通、可通关）
```

## 三、各指标的理想方向

| 指标 | 理想方向 | 原因 |
|------|----------|------|
| dp | 高 | 最优策略能拿到高分，说明迷宫有价值可挖 |
| greedy | 低（负） | 盲目贪心会亏损，说明迷宫有陷阱设计 |
| regret | 高 | 区分度大，需要动脑才能通关，策略深度好 |
| topo difficulty | 适中偏高 | 路径越复杂，迷宫越有探索性 |
| best fitness | 高 | 综合评分，优化目标 |

## 五、为什么不适合 GPU 加速

- GA 是种群迭代搜索，单次适应度评估是图遍历（非矩阵运算），并行度有限
- RL 使用 Q-Learning 查表法，不含神经网络，无大规模张量计算
- 瓶颈在 CPU 的组合搜索，GPU 加速收益极小
