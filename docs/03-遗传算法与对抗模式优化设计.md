### 1.2 染色体设计

每个染色体代表一个完整的"迷宫 + 资源布局"方案：

struct Chromosome {

### 1.3 GA 主循环

算法的核心流程 `MazeOptimizer::run()`（`maze_optimizer.cpp:25-99`）：

```
1. 初始化种群（populationSize 个随机染色体）
2. 评估所有染色体的适应度
3. 找出最优染色体作为精英
4. 循环 generations 代：
   4a. 精英保留：直接将上一代最优染色体复制到下一代
   4b. 选择：锦标赛选择两个父代
   4c. 交叉：按 crossoverRate 概率对父代进行交叉，否则直接复制父代1
   4d. 变异：按 mutationRate 概率对子代进行变异
   4e. 评估子代适应度
   4f. 将子代加入下一代种群
   4g. 更新当前代的最优染色体
   4h. 发出 generationFinished 信号（更新 UI 进度）
5. 发出 finished 信号（返回最优迷宫）
```

关键代码：

```cpp
void MazeOptimizer::run() {
    stopped_ = false;
    rng_.seed(config_.seed);

    // 1. 初始化种群
    QVector<Chromosome> population(config_.populationSize);
    for (int i = 0; i < config_.populationSize; ++i) {
        population[i] = randomChromosome(i, static_cast<quint32>(config_.seed + i));
        evaluateFitness(population[i]);
    }

    // 2. 找出初始最优
    Chromosome best = *std::max_element(population.begin(), population.end(),
        [](const Chromosome &a, const Chromosome &b) { return a.fitness < b.fitness; });

    // 3. 逐代进化
    for (int gen = 0; gen < config_.generations && !stopped_; ++gen) {
        QVector<Chromosome> nextGen;
        nextGen.reserve(config_.populationSize);

        // ★ 精英保留（Elitism = 1）
        nextGen.append(best);

        // 填充剩余位置
        while (nextGen.size() < config_.populationSize) {
            // ★ 锦标赛选择
            Chromosome parent1 = tournamentSelect(population);
            Chromosome parent2 = tournamentSelect(population);

            // ★ 交叉
            std::uniform_real_distribution<double> dist(0.0, 1.0);
            Chromosome child;
            if (dist(rng_) < config_.crossoverRate) {
                child = crossover(parent1, parent2);
            } else {
                child = parent1;  // 无性繁殖：直接复制
            }

            // ★ 变异
            if (dist(rng_) < config_.mutationRate) {
                mutate(child);
            }

            evaluateFitness(child);
            nextGen.append(child);
        }

        population = nextGen;
        // ...更新 best，发出进度信号...
    }
    emit finished(best.maze);
}
```

### 1.4 初始种群生成

`randomChromosome()`（`maze_optimizer.cpp:110-137`）负责创建初始染色体：

```
初始化流程：
1. 确定基础算法（混合模式轮询四种 / 单模式固定一种）
2. 用该算法生成迷宫拓扑（maze.generate）
3. 根据配置放置资源：
   - 对抗模式 → ResourcePlacer::placeAdversarial()
   - 智能模式 → ResourcePlacer::placeSmart()
   - 随机模式 → maze.placeResources()
```

混合模式下算法轮询的实现：

```cpp
MazeAlgorithm MazeOptimizer::algorithmForIndex(int index) {
    static constexpr MazeAlgorithm kAllAlgorithms[4] = {
        MazeAlgorithm::DivideAndConquer,   // index 0,4,8,...
        MazeAlgorithm::KruskalMst,         // index 1,5,9,...
        MazeAlgorithm::DepthFirstSearch,   // index 2,6,10,...
        MazeAlgorithm::BreadthFirstSearch  // index 3,7,11,...
    };
    return kAllAlgorithms[((index % 4) + 4) % 4];
}
```

**设计意图**：混合模式增加了初始种群的多样性。如果只使用一种算法，所有初始个体的拓扑结构偏向相同，GA 可能陷入局部最优；混合四种算法提供了四种不同"风格"的初始拓扑，增加了搜索空间的覆盖面。

### 1.5 锦标赛选择

`tournamentSelect()`（`maze_optimizer.cpp:390-401`）：

```cpp
Chromosome MazeOptimizer::tournamentSelect(const QVector<Chromosome> &population) {
    std::uniform_int_distribution<int> dist(0, population.size() - 1);
    const Chromosome *best = nullptr;
    for (int i = 0; i < config_.tournamentSize; ++i) {  // 默认 tournamentSize=3
        const Chromosome &candidate = population[dist(rng_)];
        if (!best || candidate.fitness > best->fitness) {
            best = &candidate;
        }
    }
    return *best;
}
```

- **可放回抽样**：同一个染色体可能被多次选中
- **tournamentSize=3**：较小，保持适中的选择压力
- 选择压力影响：太小 → 收敛慢；太大 → 过早收敛（premature convergence）

### 1.6 交叉操作

GA 的交叉操作是本项目最重要的设计之一。由于迷宫是**生成树**（不是一般图），普通的单点交叉或均匀交叉会破坏生成树性质。本项目使用了一种**保证生成树性质**的交叉方法：

```cpp
Chromosome MazeOptimizer::crossover(const Chromosome &a, const Chromosome &b) {
    const int totalCells = config_.rows * config_.columns;
    const int targetEdges = totalCells - 1;

    // 1. 收集两个父代的所有边
    QVector<MazeEdge> edgesA = a.maze.allEdges();
    QVector<MazeEdge> edgesB = b.maze.allEdges();

    // 2. 合并并打乱
    QVector<MazeEdge> combined = edgesA + edgesB;
    std::shuffle(combined.begin(), combined.end(), rng_);

    // 3. 使用并查集从合并边集中构造一棵生成树
    //    （类似 Kruskal 算法：接受连接不同分量的边）
    //    ...

    // 4. 如果合并的边不够 V-1 条（罕见情况），
    //    从左到右、从上到下补足剩余边
    for (int cell = 0; childEdges.size() < targetEdges; ++cell) {
        // 尝试添加 cell→right 和 cell→down 的边
        // 只接受连接不同分量的边
    }

    // 5. 用选中的边重建迷宫
    child.maze.setFromEdges(config_.rows, config_.columns, childEdges, rng_());

    // 6. 重新放置资源（对抗/智能/随机）
    // ...
}
```

**算法图解**：

```
父代A的边:  ┌─┬─┐    父代B的边:  ┌──┬─┐
           ├─┼─┤              ├──┼─┤
           └─┴─┘              └──┴─┘

合并+打乱: {A边1, B边1, A边3, B边2, A边2, B边3, ...}

用并查集依次接受连接不同分量的边 → 子代生成树（继承了父代A和B的基因）
```

**关键特性**：
- 子代**一定是合法完美迷宫**（生成树）：并查集保证不产生环
- 子代的边来自两个父代，实现了基因混合
- 补足机制保证即使极端情况下也能生成完整迷宫

### 1.7 变异操作

变异有两种策略，随机选择（`maze_optimizer.cpp:258-388`）：

**策略一：完全重新生成（30% 概率）**

```cpp
if (dist(rng_) < 0.3) {
    // 用随机算法（或基础算法）重新生成整个迷宫
    const MazeAlgorithm algo = config_.useMixedAlgorithms
        ? algorithmForIndex(rng_() % 4)     // 混合模式：随机选择一种
        : config_.baseAlgorithm;             // 单模式：固定基础算法
    chrom.maze.generate(config_.rows, config_.columns, algo, newSeed);
    // 重新放置资源...
}
```

这是一种"大变异"——完全改变拓扑结构，帮助 GA 跳出局部最优。

**策略二：边交换变异（70% 概率）**

边交换是保持完美迷宫性质的核心变异操作：

```
边交换步骤：
1. 找到所有"墙"（不在当前生成树中的候选边）
2. 随机选一堵墙凿开 → 加入这条边会在生成树中产生一个唯一的环
3. BFS 找到这个环
4. 从环中随机删除一条边（不能是刚加入的那条）
   → 恢复生成树性质（仍然是 V-1 条边，连通，无环）
```

代码实现：

```cpp
// 1. 找到所有墙候选
QVector<MazeEdge> wallCandidates;
for (int cell = 0; cell < totalCells; ++cell) {
    // 检查 cell→right 和 cell→down 是否不存在于当前边集中
}

// 2. 随机选一堵墙
const MazeEdge newEdge = wallCandidates[random];

// 3. BFS 找到 newEdge 两端在原迷宫中的路径（即形成的环）
QVector<int> parent(totalCells, -1);
QQueue<int> queue;
parent[newEdge.from] = newEdge.from;
queue.enqueue(newEdge.from);
while (!queue.isEmpty() && parent[newEdge.to] < 0) {
    int cur = queue.dequeue();
    for (int next : chrom.maze.neighbors(cur)) {
        if (parent[next] < 0) {
            parent[next] = cur;
            queue.enqueue(next);
        }
    }
}

// 4. 回溯得到环路径
QVector<int> cyclePath;
for (int cell = newEdge.to;; cell = parent[cell]) {
    cyclePath.append(cell);
    if (cell == newEdge.from) break;
}

// 5. 随机删除环中的一条边（不删除刚加入的 newEdge）
//    由于 cyclePath 是 BFS 路径，newEdge.to 到 newEdge.from，
//    所以删除环中索引 i 到 i+1 的边

// 6. 更新边集
newEdgeSet.remove(edgeKey(removeA, removeB));
newEdgeSet.insert(edgeKey(newEdge.from, newEdge.to));
chrom.maze.setFromEdges(rows, cols, newEdges, rng_());
```

**图解：边交换变异**

```
变异前：            加入墙(2,5)：        删除环中边(4,5)：
┌─┬─┬─┐           ┌─┬─┬─┐           ┌─┬─┬─┐
│0─1─2│           │0─1─2│           │0─1─2│
├─┼─┼─┤           ├─┼─┼─┤           ├─┼─┼─┤
│3 4 5│     →     │3 4─5│     →     │3 4 5│
└─┴─┴─┘           └─┴─┴─┘           └─┴─┴─┘
                  形成环 2-1-0-3-4-5 删除 4-5 的边
                  （BFS路径就是环）   恢复生成树
```

两种变异策略结合：
- 30% 完全重生成：提供**探索（exploration）**能力
- 70% 边交换：提供**局部搜索（exploitation）**能力

### 1.8 精英保留

每代只保留 1 个最优个体（Elite = 1），直接复制到下一代：

```cpp
nextGen.append(best);  // 精英直接存活到下一代
```

精英数为 1 是一个保守的选择——保留了最优解但不过度压制探索。协同进化中（CoEvolution）精英数设为 `max(2, popSize/5)`，更加激进。

## 2. 适应度评估（MazeEvaluator）

适应度评估是整个优化系统的**核心裁判**——它定义了什么是"好迷宫"。

### 2.1 EvalResult 评估结果

```cpp
struct EvalResult {
    // 基础指标
    int dpScore = 0;              // DP 最优解得分
    int worstGreedyScore = 0;     // 最差贪心 AI 得分
    int bestGreedyScore = 0;      // 最好贪心 AI 得分
    int rlScore = 0;              // RL AI 得分（可选）

    // AI 性能指标
    double coinMissRate = 0.0;    // 金币错失率
    double trapHitRate = 0.0;     // 陷阱触发率
    double pathInefficiency = 0.0;// 路径低效率
    double topoDifficulty = 0.0;  // 拓扑难度 (0~5)

    // 设计质量指标（三维度）
    double designDiscrimination = 0.0;  // D：区分度
    double designStability = 0.0;       // C：稳定性（完成率）
    double designBalance = 0.0;         // B：平衡性

    // 综合评分
    double designGroupScore = 0.0; // 设计组综合分 (60~100)
    double finalFitness = 0.0;     // 最终适应度 = designGroupScore
};
```

### 2.2 评估流程

```

  │

  │

  │

```

### 2.3 传统适应度 vs 增强适应度

项目支持两种适应度计算模式：

**传统模式**（`useEnhancedFitness = false`）：

```
fitness = dpScore - worstGreedyScore
         = DP 最优得分 - 最差贪心AI得分
         = "遗憾值"（Regret）
```

这是一个简单直观的指标：好迷宫应该让 DP 最优解和贪心 AI 的表现差距很大——即**能"骗到"AI 玩家**。

**增强模式**（`useEnhancedFitness = true`，默认）：

```
最终 fitness = designGroupScore（一个 60~100 的分数）
```

增强模式使用多维度评估，下文详述。

### 2.4 三维度设计质量指标


#### D：区分度（Discrimination）

衡量不同 AI 策略在迷宫上的表现**差异程度**。如果四种策略的得分都一样，说明迷宫"太简单"或"太随机"——没有区分度。

```cpp
// 四种策略的资源/DP 比值
aiResourceRatios[i] = clamp01(AI资源得分 / DP最优得分)

// 计算比值的标准差
resourceStddev = stddevOf(aiResourceRatios)

// D = 标准差 / 0.35，限制在 [0, 1]
D = clamp01(resourceStddev / 0.35)
```

- $D \to 1$：四种策略表现差异大 → 迷宫有区分度 ✓
- $D \to 0$：四种策略表现差不多 → 迷宫无区分度 ✗

#### C：稳定性（Stability / Completion）

衡量有多少 AI 策略**能走到终点**。如果所有策略都中途卡死，说明迷宫"太不友好"。

```cpp
C = clamp01(到达终点的策略数 / 4.0)
```

- $C = 1$：四种策略都走到了终点 → 迷宫可完成 ✓
- $C = 0$：没有策略能走到终点 → 迷宫过于困难 ✗

#### B：平衡性（Balance）

衡量 AI 的平均表现是否接近一个"理想挑战水平"（目标均值 0.45）。

```cpp
meanResourceRatio = meanOf(aiResourceRatios)     // 平均资源比值
targetMeanRatio = 0.45                            // 目标值
targetSpread = 0.18                               // 容忍范围
B = clamp01(1.0 - abs(meanResourceRatio - 0.45) / 0.18)
```

- $B \to 1$：平均表现接近 0.45 → 既不简单也不不可能 ✗
- $B \to 0$：平均表现太高或太低 → 太简单或太难 ✗

#### 设计组综合分

三者的几何平均数映射到 60~100 区间：

```cpp
geometric = ∛(max(0, D × C × B))
designGroupScore = 60.0 + geometric × 40.0
```

**使用几何平均数而非算术平均**的原因：几何平均对"短板"更敏感。如果 $D=0.9, C=0.9, B=0.1$（平衡性差），几何平均 ≈ 0.43，远低于算术平均的 0.63。这迫使三个维度都保持较好水平。

**分数区间**：60（三个维度全为 0）到 100（三个维度全为 1）。

### 2.5 拓扑难度

`computeTopoDifficulty()` 基于迷宫的结构统计特征直接评估难度：

```cpp
double MazeEvaluator::computeTopoDifficulty(const MazeModel &maze) {
    MazeStatistics stats = maze.statistics();

    double score = 0.0;

    // 死胡同比例 → 最高贡献 2.0
    double deadEndRatio = stats.deadEnds / maze.cellCount();
    score += deadEndRatio * 2.0;

    // 最长走廊（归一化到 10） → 最高贡献 0.5
    double corridorNorm = min(1.0, stats.longestCorridor / 10.0);
    score += corridorNorm * 0.5;

    // 解路径 vs 最短路径的比值 → 额外难度
    // solutionLength / shortestPath - 1 → 最高贡献 0.5×额外
    score += (pathRatio - 1.0) * 0.5;

    // 交叉口比例 → 最高贡献 1.0
    double junctionRatio = stats.junctions / maze.cellCount();
    score += junctionRatio * 1.0;

    return clamp(score, 0.0, 5.0);
}
```

四个维度的权重解释了设计者对于"难迷宫"的定义：

| 特征 | 贡献上限 | 含义 |
|---|---|---|
| 死胡同比例 ×2.0 | 2.0 | 死胡同多 → 探索成本高 |
| 最长走廊 ×0.5 | 0.5 | 走廊长 → 路径延伸 |
| 路径绕路比 ×0.5 | 无上限 | 解路径比最短路径长 → 强迫绕路 |
| 交叉口比例 ×1.0 | 1.0 | 分支点多 → 决策点多 |

## 3. 贪心 AI 玩家（四种策略）

适应度评估依赖四种贪心 AI 策略来模拟"不同水平的玩家"。

### 3.1 玩家机制

所有贪心 AI 共享同一套底层机制：

```
· 最大步数 = cellCount × 10（对于 15×15 迷宫 = 2250 步）
· 视野 = 紧邻的 4 个方向（被墙挡住的方向看不见）
· 行动 = 每步只能移动到直接相邻的已连通格子
· 收集 = 踏上格子的瞬间收集该格的资源（金币+50 / 陷阱-30）
· BFS 寻路 = 在已知的迷宫拓扑中用 BFS 计算到目标的最短路径
· 防卡死 = 记录每格的访问次数，无资源可见时优先去最少访问的格子
```

### 3.2 四种策略详情

| 策略 | 枚举值 | 目标函数 | 行为 |
|---|---|---|---|
| **ValuePerStep** | `ValuePerStep` | $\max(val / dist)$ | 选价值/距离比最大的可见资源。会主动踩陷阱 |
| **NearestFirst** | `NearestFirst` | $\min(dist)$ | 选距离最近的可见资源。也会踩陷阱 |
| **AvoidTraps** | `AvoidTraps` | $\max(val / dist), val > 0$ | 同 ValuePerStep 但跳过负价值资源（陷阱） |
| **EndGoalFirst** | `EndGoalFirst` | — | 忽略所有资源，BFS 直达终点 |

关键代码逻辑（`greedy_player.cpp:115-158`）：

```cpp
if (strategy == GreedyStrategy::EndGoalFirst) {
    target = maze.endCell();  // 直奔终点
} else {
    // 扫描可见格子
    for (const auto &vc : visible) {
        if (vc.value == 0) continue;                    // 空格子跳过
        if (vc.value < 0 && strategy == AvoidTraps) continue; // AvoidTraps 跳过陷阱
        double sc = 0.0;
        switch (strategy) {
        case ValuePerStep: sc = vc.value / vc.dist; break;    // 价值/距离
        case NearestFirst: sc = 1.0 / vc.dist;     break;    // 1/距离
        case AvoidTraps:   sc = vc.value / vc.dist; break;    // 同 ValuePerStep 但 ≤0 跳过
        }
        // 选最优目标
    }
    // 如果没有任何资源可见 → 探索：去最少访问的邻居
}
```

### 3.3 为什么用四种策略？

四种策略模拟了**不同风格的玩家**：

- **ValuePerStep**：追求效率但短视——可能为高金币走入陷阱密集区
- **NearestFirst**：保守但贪婪——看到什么拿什么
- **AvoidTraps**：谨慎——会主动规避陷阱，但可能错过被陷阱守卫的金币
- **EndGoalFirst**：只求通关——最极端的玩家类型

评估时取**最差**的那个 AI 的得分作为 `worstGreedyScore`——因为"好迷宫"应该在最弱 AI 和最聪明 AI 之间产生明显的表现差距。

## 4. 对抗式资源放置（ResourcePlacer）

对抗模式是项目中"对抗"二字的直接体现。它不随机放资源，而是**有策略地布置资源以最大化 AI 的犯错概率**。

### 4.1 智能放置（Smart Placement）

智能放置追求**合理且有教育意义**的布局：

```
金币放置：
  1. 死胡同  → 50%（奖励深度探索）
  2. 分支深处 → 25%（branchDepth ≥ 2，激励离开主路）
  3. 剩余格子 → 25%（随机补充）

陷阱放置：
  1. 非主路交叉口 → 40%（惩罚错误的路径选择）
  2. 主路径上     → 30%（增加必经之路的挑战）
  3. 剩余格子     → 30%（随机）
```

对应代码（`resource_placer.cpp:30-98`）：

```cpp
// 死胡同金币
int deadEndCoins = min(coinsLeft * config.deadEndCoinRatio, deadEnds.size());
// 分支深处金币
int branchCoins = min(coinsLeft / 2, branchDeep.size());
// 交叉口陷阱
int junctionTraps = min(trapsLeft * config.branchTrapRatio, junctions.size());
// 主路径陷阱
int mainTraps = min(trapsLeft * config.mainPathTrapRatio, mainPathCells.size());
```

### 4.2 对抗放置（Adversarial Placement）

对抗放置追求**最大化 DP-AI 差距**——即"让 AI 表现尽可能差，但 DP 最优解仍然能拿到高分"。

代码中的中文注释原文（`resource_placer.cpp:120-146`）：

```
// === 对抗式陷阱分布 ===
// 陷阱放在分叉口（迫使AI在探索时踩到）和主路径上

// === 对抗式金币分布 ===
// 金币放在远离主路径的深处（迫使AI走冤枉路）
// 不放死胡同（太容易拿），放分支深处
```

**陷阱策略：分叉口饱和**

```
70% 陷阱 → 交叉口（包括主路径和非主路径上的所有分叉口）
30% 陷阱 → 随机位置
```

**为什么是交叉口？** 交叉口是决策点——AI 必须选择走哪个分支。如果把陷阱放在交叉口上，AI 在探索不同分支时会反复踩到，大幅降低其总收益。同时 DP 最优解可以"一次性"经过交叉口，受到的影响较小。

**金币策略：分支深处（非死胡同）**

```
第一优先：branchDepth ≥ 3 的极深分支
第二优先：branchDepth ≥ 2 的一般分支深处
第三优先：随机
```

**为什么不是死胡同？** 死胡同太容易被贪心 AI 发现——走到尽头就能拿到金币。放在分支深处意味着 AI 必须"专门绕路"走完整个分支才能拿到。贪心 AI 的近视策略可能根本不会走那么深，而 DP 最优解会精确判断"这个分支值不值得绕路"。

### 4.3 智能 vs 对抗 对比

| 维度 | 智能放置 | 对抗放置 |
|---|---|---|
| 设计目标 | 合理、教学性强的布局 | 最大化 AI 困惑程度 |
| 金币位置 | 死胡同（易于探索者） | 分支深处（需要绕路） |
| 陷阱位置 | 交叉口 40% + 主路 30% | 交叉口 70%（饱和） |
| 对一般玩家的影响 | 中等 | 较难 |
| DP-AI 遗憾值 | 中等 | 较大 |
| 是否适合作为优化目标 | 是（稳定基线） | 是（激进优化） |

## 5. 协同进化（CoEvolution）

### 5.1 核心思想

协同进化模拟了一个**军备竞赛**：

```
GA 进化迷宫 → 目标是让 RL 玩家得分更低
RL 训练玩家 → 目标是在当前最难的迷宫上表现更好
    ↑                       ↓
    └────── 交替循环 ─────────┘
```

### 5.2 配置参数

```cpp
struct CoEvolConfig {
    int cycles = 10;                  // 协同进化轮次
    int gaGenerations = 20;           // 每轮 GA 代数
    int gaPopulation = 20;            // GA 种群大小
    double gaMutationRate = 0.3;      // GA 变异率（较高）
    int rlTrainEpisodes = 100;        // RL 每轮训练回合数
    int rlTopK = 5;                   // 用最难的 K 个迷宫训练 RL
    int mazeRows = 15, mazeCols = 15;
    MazeAlgorithm baseAlgorithm = MazeAlgorithm::KruskalMst;
    bool useMixedAlgorithms = true;
    bool useSmartPlacement = true;
};
```

### 5.3 算法流程

```cpp
CoEvolResult CoEvolution::run(const CoEvolConfig &config) {
    // 初始化：生成种群，每个个体做 5 次随机边交换增加多样性
    QVector<MazeModel> population(popSize);
    for (int i = 0; i < popSize; ++i) {
        population[i].generate(rows, cols, algo, seed);
        for (int m = 0; m < 5; ++m) {
            MazeOptimizer::edgeSwap(population[i], rng);  // 初始扰动
        }
    }

    // 适应度 = DP 最优值 - RL 玩家得分（即 RL 遗憾值）
    auto evalRLFitness = [&](MazeModel &maze) -> int {
        int dpVal = maze.optimalResourceWalk().maxValue;
        RLPlayResult rl = rlPlayer.play(maze, rlCfg);
        return dpVal - rl.totalResource;  // 越大 = 迷宫越"难"倒 RL
    };

    for (int cycle = 0; cycle < config.cycles; ++cycle) {
        // ===== GA 阶段：进化迷宫（gaGenerations 代）=====
        for (int gen = 0; gen < config.gaGenerations; ++gen) {
            // 按适应度排序
            // 精英保留 top-N（N = popSize/5）
            // 锦标赛选择（只在 top half 中选）
            // 交叉 + 变异
            // 评估适应度
        }

        // ===== RL 训练阶段 =====
        // 选出 top-K 个最难的迷宫（按 RL 适应度）
        // RL 玩家在这些迷宫上训练 rlTrainEpisodes 局
        rlPlayer.trainOnMazes(hardestMazes, rlCfg);

        // 记录本轮结果
    }
}
```

### 5.4 与普通 GA 的区别

| 维度 | 普通 GA（MazeOptimizer） | 协同进化（CoEvolution） |
|---|---|---|
| 优化目标 | 击败贪心 AI | 击败 RL 玩家 |
| 适应度函数 | designGroupScore（多维度） | DP - RL（单一遗憾值） |
| 精英保留 | 1 个 | popSize/5 个（更强） |
| 选择范围 | 全种群 | 只选 top half |
| 变异率 | 0.15 | 0.3（更高探索） |
| AI 对手 | 静态（固定四种策略） | 动态（每轮训练进化） |
| 初始扰动 | 无 | 5 次边交换 |

## 6. Q-Learning 拓扑优化器

### 6.1 定位

`QLearningOptimizer` 是一个**细粒度**优化工具——它在 GA 已经产生的好迷宫基础上，通过 Q-Learning 学习哪些边交换操作能进一步增加 DP-AI 差距。

### 6.2 状态空间（75 个状态）

将迷宫的特征离散化为三个维度：

```cpp
// f1: 死胡同比例 (0~4)
f1 = min(4, deadEnds * 5 / totalCells)

// f2: 解路径长度比例 (0~4)
f2 = min(4, solutionLength * 5 / directDist)

// f3: 最优路径上陷阱比例 (0~2)
f3 = min(2, trapOnPath * 3 / totalTraps)

// 状态编码
state = f1 * 15 + f2 * 3 + f3  // 范围: 0~74 (75个状态)
```

### 6.3 动作空间

动作是**边交换操作**（同 GA 变异策略二），最多 10 个候选。

### 6.4 奖励函数

```
reward = regret(after) - regret(before)
regret = dpMaxValue - greedyRemainingResource
```

即：如果一次边交换**增加了** DP-AI 差距，获得正奖励。

## 8. 设计决策总结

### 8.1 为什么交叉操作使用并查集？

保证子代是合法完美迷宫。如果用一般的位交叉（例如从父代 A 取前半部分边，父代 B 取后半部分边），会导致环或不连通。

### 8.2 为什么适应度是 designGroupScore 而非单纯的 regret？

单纯的 `dpScore - greedyScore` 可能产生"极端迷宫"——AI 永远走不到终点的高分迷宫。这会牺牲可玩性。D、C、B 三维度强制了"好的迷宫设计"——既要难倒短视的 AI，也要保证合理的可完成性和玩家间的平衡表现。

### 8.3 为什么对抗模式陷阱放在交叉口？

交叉口是 AI 决策的关键点。一个 AI 可能在探索不同分支时反复经过同一个交叉口——每次都踩到陷阱。DP 最优解可以只经过一次，所以 DP 和 AI 的差距在交叉口陷阱上被放大。

### 8.4 为什么有 Q-Learning 优化器还要 GA？

GA 是全局搜索（改变拓扑结构的大手术），Q-Learning 是局部精调（边交换的小手术）。两者互补。

### 8.5 混合算法 vs 单算法的权衡

- **混合算法**：初始种群多样性高 → 全局搜索更充分，但每代平均质量可能较低
- **单算法**：初始种群质量高（如果用好的基础算法），但可能陷入局部最优

UI 选择了单算法（`useMixedAlgorithms = false`），可能是因为用户体验更好——用户选什么算法就优化什么算法。命令行模式默认混合算法。
