## 0. 前置知识：什么是"完美迷宫"

本项目生成的迷宫是**完美迷宫（Perfect Maze）**。在计算机科学中，完美迷宫定义为：

- 所有格子（顶点）相互连通
- 任意两个格子之间有且仅有一条简单路径
- 等价于：网格图的**一棵生成树（Spanning Tree）**

简单说：**全连通 + 无环路 = 完美迷宫**。

项目的合法性校验函数 `validatePerfect()`（`maze.cpp:460-502`）正是按这个标准来检查的：

```cpp
// 伪代码逻辑
reached = BFS从起点能到达的格子数
edgeCount = 迷宫中的通道边数
valid = (reached == V) && (edgeCount == V - 1)
// 即：全部可达 且 E = V - 1
```

四种算法虽然实现思路完全不同，但最终结果都是构造一棵包含所有格子的生成树。

## 0.1 公共数据模型

所有四种算法共享同一套底层数据结构，定义在 `src/include/core/maze.h` 中：

```cpp
class MazeModel {
    int rows_, columns_;                     // 迷宫逻辑行/列数（如 15×15）
    QVector<std::array<bool, 4>> passages_;  // 每格4个方向的通道：上(0) 右(1) 下(2) 左(3)
    QVector<int> resources_;                 // 每格上的资源（金币+50 / 陷阱-30 / 无0）
    QVector<MazeEdge> generationSteps_;      // 开墙顺序（用于动画回放）
    std::mt19937 random_;                    // Mersenne Twister 随机数生成器
    int startCell_, endCell_, bossCell_;    // 起点、终点、BOSS格子编号
};
```

**核心辅助方法：**

| 方法 | 功能 |
|---|---|
| `index(row, col)` | 将二维坐标转换为一维线性索引 `row * columns + col` |
| `rowOf(cell)` / `columnOf(cell)` | 一维索引还原为二维坐标 |
| `gridNeighbors(cell)` | 返回格子在网格中的上下左右邻居（不论是否有通道） |
| `neighbors(cell)` | 返回当前已开通的邻居（用于路径搜索等） |
| `carve(first, second)` | 在两个相邻格子之间开一扇门（双向设置通道位，并记录到 `generationSteps_`） |

**`carve()` 的实现细节**（`maze.cpp:157-176`）：

```cpp
void MazeModel::carve(int first, int second) {
    // 根据 first 到 second 的行列偏移，判断方向
    const int rowDelta = rowOf(second) - rowOf(first);
    const int columnDelta = columnOf(second) - columnOf(first);
    int direction = -1;
    if (rowDelta == -1 && columnDelta == 0)      direction = Up;      // second 在 first 上方
    else if (rowDelta == 0 && columnDelta == 1)  direction = Right;   
    else if (rowDelta == 1 && columnDelta == 0)  direction = Down;    
    else if (rowDelta == 0 && columnDelta == -1) direction = Left;    

    passages_[first][direction] = true;             // first → second 方向打开
    passages_[second][(direction + 2) % 4] = true;  // second → first 方向也打开（反方向 = dir+2 mod 4）
    generationSteps_.append({first, second});       // 记录开墙步骤，供动画播放
}
```

**`gridNeighbors()` 的实现**（`maze.cpp:138-155`）：

```cpp
QVector<int> MazeModel::gridNeighbors(int cell) const {
    QVector<int> result;
    const int row = rowOf(cell), column = columnOf(cell);
    if (row > 0)                result.append(index(row - 1, column));  // 上
    if (column + 1 < columns_)  result.append(index(row, column + 1));  // 右
    if (row + 1 < rows_)        result.append(index(row + 1, column));  // 下
    if (column > 0)             result.append(index(row, column - 1));  // 左
    return result;
}
```

## 0.2 统一入口函数 `generate()`

所有算法通过同一个入口函数 `MazeModel::generate()` 调度（`maze.cpp:74-92`）：

```cpp
void MazeModel::generate(int rows, int columns, MazeAlgorithm algorithm, quint32 seed) {
    reset(rows, columns, seed);    // 清空数据，初始化尺寸，设置随机种子
    switch (algorithm) {
    case MazeAlgorithm::DivideAndConquer:

        break;
    case MazeAlgorithm::KruskalMst:
        generateKruskal();
        break;
    case MazeAlgorithm::DepthFirstSearch:
        generateDepthFirst();
        break;
    case MazeAlgorithm::BreadthFirstSearch:
        generateBreadthFirst();
        break;
    }
    chooseDiameterEndpoints();  // 后处理：找树直径端点作为起点和终点
    hasBoss_ = true;
}
```

`reset()` 负责把 `passages_` 全部清零（四面都是墙）、设置 `startCell_ = 0`、`endCell_ = cellCount() - 1`、初始化随机数引擎。

## 1. 算法一：分治法（Divide and Conquer）

### 1.1 基本原理

分治法是一种**递归区域划分**的迷宫生成算法。核心思想：

1. 将一个矩形区域沿一条直线**一分为二**
2. 递归地对两个子区域分别生成迷宫（各自是一棵树）
3. 在两个子区域之间的墙上**随机开一个洞**，将它们连通

由于每个子区域内部已是树（无环），两个树之间只加一条边，最终合并后的图仍然是树——保证了完美迷宫的性质。

### 1.2 代码详解

```cpp
void MazeModel::generateDivideAndConquer(int top, int bottom, int left, int right) {
    // 递归 base case：区域缩小到单个格子，无需处理
    if (top == bottom && left == right) {
        return;
    }

    const int height = bottom - top + 1;   // 当前区域高度（格子数）
    const int width  = right - left + 1;   // 当前区域宽度（格子数）

    if (width >= height && width > 1) {
        // ─── 垂直分割 ───
        // 随机选一列作为分割线（范围 [left, right-1]）
        std::uniform_int_distribution<int> splitDistribution(left, right - 1);
        const int split = splitDistribution(random_);

        // 递归处理左右两个子区域
        generateDivideAndConquer(top, bottom, left, split);        // 左半区
        generateDivideAndConquer(top, bottom, split + 1, right);   // 右半区

        // 在分割线上随机选一行，打开一个通道连接左右两个区域
        std::uniform_int_distribution<int> openingDistribution(top, bottom);
        const int openingRow = openingDistribution(random_);
        carve(index(openingRow, split), index(openingRow, split + 1));

    } else {
        // ─── 水平分割 ───
        // 随机选一行作为分割线（范围 [top, bottom-1]）
        std::uniform_int_distribution<int> splitDistribution(top, bottom - 1);
        const int split = splitDistribution(random_);

        // 递归处理上下两个子区域
        generateDivideAndConquer(top, split, left, right);         // 上半区
        generateDivideAndConquer(split + 1, bottom, left, right);  // 下半区

        // 在分割线上随机选一列，打开一个通道连接上下两个区域
        std::uniform_int_distribution<int> openingDistribution(left, right);
        const int openingColumn = openingDistribution(random_);
        carve(index(split, openingColumn), index(split + 1, openingColumn));
    }
}
```

### 1.3 执行过程图解

假设一个 4×4 的迷宫，递归过程如下：

```
第一层：width=4 >= height=4, 垂直分割
┌─┬─┬─┬─┐      ┌─┬─┰─┬─┐
│ │ │ │ │      │ │ │ │ │ │
├─┼─┼─┼─┤  →  ├─┼─╂─┼─┤   split=1, 在 row=2 处开通道
│ │ │ │ │      │ │ ┊ │ │ │
├─┼─┼─┼─┤      ├─┼─╂─┼─┤
│ │ │ │ │      │ │ │ │ │ │
└─┴─┴─┴─┘      └─┴─┸─┴─┘
               左(0-1列)  右(2-3列)

第二层（左右各自递归）：
左区 2×4: height=2 < width=4, 再垂直分割
右区 2×4: height=2 < width=4, 再垂直分割

第三层：宽度=1，改为水平分割
...最终每个子区域都变成单格才停止
```

### 1.4 关键特性

| 维度 | 分析 |
|---|---|
| **分割方向选择** | `width >= height` 优先垂直分割。这保证了区域尽量接近正方形，避免产生极端狭长的迷宫 |
| **唯一通道保证** | 每次分割只产生一个连通口，子区域之间只有一个通道，杜绝环路的产生 |
| **递归深度** | 平衡时 $O(\log V)$，最坏时可能退化到 $O(V)$（例如始终沿同一方向分割） |
| **时间复杂度** | $O(V)$——每个格子被递归访问恰好一次 |
| **空间复杂度** | $O(\log V)$ 最优递归栈；$O(V)$ 最坏情况 |
| **迷宫风格** | 有明显的**矩形分区痕迹**，从整体上看像是大的矩形区域之间仅有狭窄通道相连 |

## 2. 算法二：Kruskal 最小生成树法（Kruskal MST）

### 2.1 基本原理

Kruskal 算法是图论中经典的**最小生成树**算法。应用到迷宫生成：

1. 将每个相邻格子之间的"墙"看作一条**候选边**
2. 随机打乱所有候选边的顺序（等价于给每条边随机赋权值）
3. 按打乱后的顺序依次处理每条边：如果这条边连接的两个格子当前属于**不同的连通分量**，就凿开这堵墙（接受这条边），并用**并查集**合并两个分量
4. 如果两个格子已经在同一连通分量中（说明凿开这堵墙会形成环路），则跳过
5. 当接受了 V-1 条边时，生成了完整生成树

### 2.2 并查集（Disjoint Set Union）实现

Kruskal 算法依赖高效的并查集数据结构。项目中的实现（`maze.cpp:17-49`）：

```cpp
class DisjointSet {
public:
    explicit DisjointSet(int size) : parent_(size), rank_(size, 0) {
        std::iota(parent_.begin(), parent_.end(), 0);  // 每个元素的父节点初始化为自己
    }

    // 查找根节点，同时路径压缩
    int find(int value) {
        if (parent_[value] != value) {
            parent_[value] = find(parent_[value]);  // 递归压缩路径
        }
        return parent_[value];
    }

    // 合并两个集合，按秩合并
    bool unite(int first, int second) {
        first  = find(first);
        second = find(second);
        if (first == second) {
            return false;  // 已在同一集合
        }
        if (rank_[first] < rank_[second]) {
            std::swap(first, second);  // 确保 first 的秩 ≥ second 的秩
        }
        parent_[second] = first;       // 将秩小的集合并入秩大的
        if (rank_[first] == rank_[second]) {
            ++rank_[first];
        }
        return true;  // 合并成功
    }

private:
    QVector<int> parent_;  // 父节点
    QVector<int> rank_;    // 秩（近似树高）
};
```

**复杂度**：`find()` 和 `unite()` 的均摊时间复杂度为 $O(\alpha(V))$，其中 $\alpha$ 是反阿克曼函数，在可观测宇宙范围内不会超过 4，可视为常数。

### 2.3 Kruskal 迷宫生成代码详解

```cpp
void MazeModel::generateKruskal() {
    // 第一步：收集所有可能的墙（候选边）
    QVector<MazeEdge> candidates;
    for (int row = 0; row < rows_; ++row) {
        for (int column = 0; column < columns_; ++column) {
            const int cell = index(row, column);
            // 每格只考虑右邻居和下邻居，保证每条边只被添加一次
            if (column + 1 < columns_) {
                candidates.append({cell, index(row, column + 1)});  // → 右边
            }
            if (row + 1 < rows_) {
                candidates.append({cell, index(row + 1, column)});  // ↓ 下边
            }
        }
    }
    // 总边数 = rows*(columns-1) + (rows-1)*columns = 2*V - rows - columns

    // 第二步：随机打乱所有边（等价于随机权重）
    std::shuffle(candidates.begin(), candidates.end(), random_);

    // 第三步：Kruskal 主循环
    DisjointSet sets(cellCount());
    for (const MazeEdge &edge : candidates) {
        if (sets.unite(edge.from, edge.to)) {  // 两端在不同集合？接受并合并
            carve(edge.from, edge.to);         // 凿开墙
        }
        // 如果在同一集合，跳过（否则会形成环）
    }
}
```

> **注意**：主项目的 Kruskal 实现没有显式的提前终止条件（它遍历全部边直到结束，但只有 V-1 条能通过 `unite` 检测）。独立提交版 `REQ_0_4` 中有 `if (edges.size() == rows * columns - 1) break;` 的优化。

### 2.4 为什么打乱边等价于随机权重？

标准的 Kruskal 算法是"按权重从小到大处理边"。如果给每条边随机分配一个 $[0,1)$ 的权值，并升序排列，那么边处理的顺序就是完全随机的。项目中用 `std::shuffle` 直接随机打乱边数组，效果完全相同，且避免了排序的 $O(E \log E)$ 开销——打乱的复杂度是 $O(E)$。

### 2.5 关键特性

| 维度 | 分析 |
|---|---|
| **时间复杂度** | $O(E \cdot \alpha(V))$，近乎线性。如果用排序+权重，则为 $O(E \log E)$ |
| **空间复杂度** | $O(V + E)$，需存储所有候选边 |
| **迷宫风格** | 分支较为**均匀均衡**。由于每条边被接受的概率相同，迷宫的结构没有明显的空间偏向 |
| **数学性质** | 在所有可能的生成树中**均匀分布**——每条边被选入生成树的概率均等 |
| **视觉效果** | 通道较"随机"，看不到明显的结构偏好 |

## 3. 算法三：回溯法 / 深度优先搜索（DFS Backtracking）

### 3.1 基本原理

对网格图做**随机化的深度优先搜索**：

1. 从一个起始格子出发，标记为已访问
2. 检查当前格子的四个邻居中哪些尚未被访问，随机选一个
3. 凿开当前格子和选中邻居之间的墙，移动到该邻居
4. 重复步骤 2-3
5. 当某个格子的所有邻居都已被访问时（走入死胡同），**回溯**到上一个还有未访问邻居的格子
6. 当回溯到起点且起点也无未访问邻居时，所有格子都被访问完毕

这个过程的本质是：**在网格图上随机走出一条深度优先生成树**。由于每个格子只在首次被访问时与"父节点"之间建立一条边，且后续回溯不会再添加边，所以结果是一棵生成树。

### 3.2 代码详解

```cpp
void MazeModel::generateDepthFirst() {
    QVector<bool> visited(cellCount(), false);
    QVector<int> stack{startCell()};    // 显式栈，初始放入起点（格子0）
    visited[startCell()] = true;

    while (!stack.isEmpty()) {
        const int current = stack.last();  // 栈顶 = 当前探索的格子

        // 收集当前格子的所有未访问邻居
        QVector<int> choices;
        for (int next : gridNeighbors(current)) {
            if (!visited[next]) {
                choices.append(next);
            }
        }

        if (choices.isEmpty()) {
            // 没有未访问邻居 → 死胡同，回溯
            stack.removeLast();
            continue;
        }

        // 随机选择一个未访问邻居
        std::shuffle(choices.begin(), choices.end(), random_);
        const int next = choices.first();

        visited[next] = true;
        carve(current, next);    // 凿开 current ↔ next 的墙
        stack.append(next);      // 前进
    }
}
```

### 3.3 执行过程示意

```
栈: [0]                    栈: [0,1]                 栈: [0,1,5]
┌───┬───┬───┐            ┌───┬───┬───┐           ┌───┬───┬───┐
│ 0 │   │   │            │ 0→│ 1 │   │           │ 0→│ 1 │   │
├───┼───┼───┤  随机选   ├───┼───┼───┤  随机选  ├───┼───┼───┤
│   │   │   │  格子1     │   │   │   │  格子5    │   │ ↓ │   │
├───┼───┼───┤     →     ├───┼───┼───┤     →    ├───┼───┼───┤
│   │   │   │            │   │   │   │           │   │ 5 │   │
└───┴───┴───┘            └───┴───┴───┘           └───┴───┴───┘

栈: [0,1]  (pop 5)        栈: [0,1,2]             ...
格子5死胡同，回溯             继续前进
```

### 3.4 关键特性

| 维度 | 分析 |
|---|---|
| **时间复杂度** | $O(V+E)$。每个格子访问一次，每条候选边考虑常数次 |
| **空间复杂度** | $O(V)$——`visited` 数组 + 递归栈（最坏情况栈深 = V） |
| **迷宫风格** | **长走廊多**，分支点少。因为 DFS 倾向于"一条路走到黑"才回头 |
| **死胡同** | DFS 生成的迷宫通常有较多的死胡同（每个回溯点对应一个死胡同分支的终结） |
| **视觉效果** | 迷宫看起来像一条蜿蜒的主路加上少量分支，空间利用率较高但路线偏长 |

### 3.5 与独立提交版的微小差异

独立提交版 `REQ_0_2` 将"查找未访问邻居"提取为独立函数 `unvisitedNeighbors()`，而主项目直接在内循环中完成筛选。此外独立版注释明确解释了正确性：

> *"A cell is connected only when first visited. Backtracking changes the search position but never adds a second parent, so the result is a tree."*

## 4. 算法四：分支限界 / 优先队列式 BFS（Branch-and-Bound BFS）

### 4.1 基本原理

这是四种算法中**最复杂**的一个。它不是简单的先进先出 BFS，而是一种**带分支限界的多层随机重启**方法。核心设计思路：

1. 用**优先队列**代替普通队列，按"下界"从小到大出队
2. 下界公式 `depth × 10000 + degree[from] × 100 + randomWeight` 保证**严格按 BFS 层次顺序**扩展（depth 项主导），同一层内再按度数择优
3. 每个候选迷宫使用**随机根节点**启动（不从固定角点开始）
4. 最多尝试 **12 次**独立生成，每次从不同随机根节点出发
5. 保留**评分最高的候选**作为最终结果，评分 = `solutionLength × 10 + deadEnds`（即更长的解路径和更多的死胡同得分更高）
6. 如果某次尝试已经达到足够长的路径（`solutionLength >= directDistance + 6`），提前终止

### 4.2 核心数据结构

```cpp
struct FrontierBranch {
    int lowerBound   = 0;   // 下界分数（越小越优先出队）
    quint32 tieBreaker = 0; // 平局时的随机决胜因子
    int from         = -1;  // 连接的源格子
    int to           = -1;  // 连接的目标格子
    int depth        = 0;   // BFS 层数
    int randomWeight = 0;   // 随机权重（0~99）
    int sourceDegree = 0;   // 入队时源格子的度数
};

struct WorseBranch {
    // 优先队列的比较器：lowerBound 小的优先，相同则 tieBreaker 小的优先
    bool operator()(const FrontierBranch &first, const FrontierBranch &second) const {
        if (first.lowerBound != second.lowerBound)
            return first.lowerBound > second.lowerBound;  // > 使得小的在堆顶
        return first.tieBreaker > second.tieBreaker;
    }
};
```

### 4.3 下界公式详解

```
lowerBound = depth × 10000 + degree[from] × 100 + randomWeight
```

这个公式是理解算法的关键：

| 项 | 权重 | 作用 |
|---|---|---|
| `depth` | ×10000 | **绝对主导**。强制所有较浅层的分支先于较深层被处理，保证严格的 BFS 层次纪律 |
| `degree[from]` | ×100 | 同层内部排序。度数较低的源格子优先，这鼓励树结构不要太"星型"（即避免一个格子连接太多邻居） |
| `randomWeight` | ×1 | 范围 [0, 99]。只影响同层同度数的兄弟分支，引入随机性 |

**为什么不直接用简单的 BFS？** 简单的先进先出 BFS 会产生以起点为中心的"星型"结构——所有格子都在距离起点 1~2 步的地方连接，迷宫会极度简单。通过优先队列下的度数惩罚和随机加权，BFS 在保持层次纪律的同时，产生了更自然的迷宫结构。

### 4.4 分支限界的"限界"在哪体现？

体现在两个层面：

**层面一：内部优先队列剪枝**

每次从优先队列取出分支后，会检查"staleness"——如果 `sourceDegree` 与入队时记录的度数不同（说明从该源格子出发的其他分支已被接受，度数增加了），则需要**重新计算下界并重新入队**。这保证了：
- 一个格子不会接受"过期"的低优先级连接
- 度数变化会导致下界改变，迫使算法重新评估

```cpp
// Staleness 检查（maze.cpp:312-318）
if (branch.sourceDegree != degree[branch.from]) {
    branch.sourceDegree = degree[branch.from];
    branch.lowerBound = branch.depth * 10000 + branch.sourceDegree * 100
                        + branch.randomWeight;
    frontier.push(branch);   // 重新入队
    continue;                // 跳过当前过期分支
}
```

**层面二：外向随机重启 + 最佳候选保留**

每次尝试完成后调用 `statistics()` 获取解路径长度和死胡同数，计算评分。12 次独立尝试中保留最高分的候选。如果某次尝试的解路径已足够长（`>= directDistance + 6`），则提前停止。

```cpp
// 提前终止条件（maze.cpp:334）
if (stats.solutionLength >= directDistance + 6) {
    break;  // 已达到足够难度，不再尝试
}
```

其中 `directDistance = rows + columns - 2`，即网格对角曼哈顿距离。例如 15×15 的迷宫，`directDistance = 28`，终止条件是解路径 ≥ 34。

### 4.5 完整代码流程

```cpp
void MazeModel::generateBreadthFirst() {
    QVector<std::array<bool, 4>> bestPassages;
    QVector<MazeEdge> bestSteps;
    int bestScore = -1;
    const int directDistance = rows_ + columns_ - 2;

    // 最多 12 次独立尝试
    for (int attempt = 0; attempt < 12; ++attempt) {
        // 每次尝试都完全重置
        passages_.fill({false, false, false, false}, cellCount());
        generationSteps_.clear();

        QVector<bool> visited(cellCount(), false);
        QVector<int>  degree(cellCount(), 0);
        std::priority_queue<FrontierBranch, QVector<FrontierBranch>, WorseBranch> frontier;

        // addBranches lambda: 将某个格子的所有未访问邻居加入优先队列
        auto addBranches = [&](int cell, int depth) {
            QVector<int> choices = gridNeighbors(cell);
            std::shuffle(choices.begin(), choices.end(), random_);  // 打乱邻居顺序
            for (int next : choices) {
                if (visited[next]) continue;
                const int randomWeight = randomCost(random_);
                const int lowerBound = depth * 10000 + degree[cell] * 100 + randomWeight;
                frontier.push({lowerBound, tieBreaker(random_), cell, next,
                               depth, randomWeight, degree[cell]});
            }
        };

        // 随机选择根节点
        const int root = rootDistribution(random_);
        visited[root] = true;
        addBranches(root, 1);  // 根节点深度为 1

        // BFS + 优先队列主循环
        while (!frontier.empty()) {
            FrontierBranch branch = frontier.top();
            frontier.pop();

            if (visited[branch.to]) continue;    // 目标已访问 → 跳过（防环）

            // Staleness 检查
            if (branch.sourceDegree != degree[branch.from]) {
                branch.sourceDegree = degree[branch.from];
                branch.lowerBound = branch.depth * 10000
                                  + branch.sourceDegree * 100 + branch.randomWeight;
                frontier.push(branch);  // 重新入队
                continue;
            }

            visited[branch.to] = true;
            carve(branch.from, branch.to);
            ++degree[branch.from];
            ++degree[branch.to];
            addBranches(branch.to, branch.depth + 1);
        }

        // 评分并保留最佳候选
        chooseDiameterEndpoints();
        const MazeStatistics stats = statistics();
        const int score = stats.solutionLength * 10 + stats.deadEnds;
        if (score > bestScore) {
            bestScore = score;
            bestPassages = passages_;
            bestSteps    = generationSteps_;
        }

        // 提前终止
        if (stats.solutionLength >= directDistance + 6) break;
    }

    // 恢复最佳候选的结果
    passages_        = bestPassages;
    generationSteps_ = bestSteps;
}
```

### 4.6 关键特性

| 维度 | 分析 |
|---|---|
| **时间复杂度** | $O(12 \times E \log E) = O(E \log E)$，优先队列操作主导 |
| **空间复杂度** | $O(E)$，优先队列最多容纳所有候选边 |
| **迷宫风格** | 保持 BFS 的层次扩展特征，但由于度数惩罚和随机加权，产生了比纯 BFS 更自然的布局。随机重启 + 最佳候选机制使其在四种方法中**解路径最长、最有挑战性** |
| **评分函数** | `solutionLength × 10 + deadEnds` 追求两点：长解路径和更多死胡同 |
| **终止策略** | 解路径达到 `(rows + cols - 2) + 6` 即认为足够好，不再尝试 |

### 4.7 与独立提交版的微小差异

独立提交版 `REQ_0_3` 中，评分函数放在 `candidateScoreAndDiameter()` 里，返回 `{diameter * 10 + deadEnds, diameter}`。主项目的代码则通过 `MazeStatistics` 获取 `solutionLength`（即树的直径长度），逻辑一致。

## 5. 公共后处理：直径端点选取

无论使用哪种算法生成迷宫，`generate()` 在最后都会调用 `chooseDiameterEndpoints()`（`maze.cpp:94-136`）：

```cpp
void MazeModel::chooseDiameterEndpoints() {
    // 第一次 BFS：从格子 0 出发，找到最远格子 → 设为起点
    startCell_ = farthestFrom(0);

    // 第二次 BFS：从起点出发，找到最远格子 → 设为终点，同时记录路径
    endCell_ = farthestFrom(startCell_);

    // 沿 BFS 记录的父指针回溯，bossCell 设为终点前一步的格子
    bossCell_ = parent[endCell_];  // 守卫在终点前一格
}
```

**设计意图**：如果将起点和终点固定在格子 0 和对角位置（如格子 0 和 V-1），对于 BFS 方法生成的迷宫，由于 BFS 自然产生较短的路径，0 到 V-1 的解路径会很短，使得迷宫过于简单。通过两次 BFS 求**树的直径**（树中最长简单路径），确保了起点到终点的距离是迷宫中最远的两点，**最大化了主路径的长度**，保证了迷宫的挑战性。

### BFS 最远格查找实现

```cpp
auto farthestFrom = [&](int source) {
    QVector<int> distance(cellCount(), -1);  // -1 表示未访问
    QQueue<int> queue;
    distance[source] = 0;
    queue.enqueue(source);
    int farthest = source;
    while (!queue.isEmpty()) {
        const int current = queue.dequeue();
        if (distance[current] > distance[farthest]) {
            farthest = current;  // 记录当前最远格子
        }
        for (int next : neighbors(current)) {
            if (distance[next] >= 0) continue;
            distance[next] = distance[current] + 1;
            queue.enqueue(next);
        }
    }
    return farthest;
};
```

## 6. 四种算法对比总结

### 6.1 复杂度对比

| 算法 | 时间复杂度 | 空间复杂度 | 核心数据结构 |
|---|---|---|---|
| 分治 (Divide & Conquer) | $O(V)$ | $O(\log V)$ 典型 / $O(V)$ 最坏 | 递归栈 |
| Kruskal MST | $O(E \cdot \alpha(V))$ ≈ $O(E)$ | $O(V+E)$ | 并查集 |
| 回溯 DFS | $O(V+E)$ | $O(V)$ | 显式栈 |
| BFS 分支限界 | $O(E \log E)$ | $O(E)$ | 优先队列 + 多次重启 |

> 注：$V = rows \times columns$ 为网格顶点数，$E \approx 2V$ 为网格边数。对于 15×15 的迷宫，$V=225$, $E \approx 420$。

### 6.2 迷宫风格对比

| 算法 | 视觉效果 | 特点 |
|---|---|---|
| 分治 | 区域分块明显 | 可看到清晰的矩形"房间"，房间之间仅有一两个窄通道连接 |
| Kruskal | 均匀随机 | 无明显空间偏向，所有可能的结构等概率出现 |
| 回溯 DFS | 长走廊 | 蜿蜒曲折的长路径，分支点较少，死胡同较多 |
| BFS 分支限界 | 分层展开 | 从根节点向外层层扩散，但经优化后自然度介于 DFS 和 Kruskal 之间 |

### 6.3 迷宫统计数据对比（15×15 迷宫，不同 seed 取均值）

| 指标 | 分治 | Kruskal | DFS | BFS (分支限界) |
|---|---|---|---|---|
| 解路径长度 | 中等 | 中等偏长 | 较长 | **最长** |
| 死胡同数量 | 中等 | 较多 | **最多** | 较多 |
| 交叉口数量 | 较少 | 中等 | 较少 | 中等 |
| 最长走廊 | 中等 | 较短 | **最长** | 中等偏长 |

### 6.4 算法选择建议

| 场景 | 推荐算法 |
|---|---|
| 需要快速生成、简单直观 | 分治 |
| 追求迷宫结构的均匀随机性 | Kruskal |
| 希望迷宫有长走廊和探索感 | DFS |
| 追求最大挑战性（最长路径、最多死路） | BFS 分支限界 |

### 6.5 在 UI 中的对应关系

在程序界面中，四种算法通过下拉框选择：

| UI 标签 | 算法 | Enum 值 |
|---|---|---|
| 分治 | Divide and Conquer | `MazeAlgorithm::DivideAndConquer` (0) |
| 贪心 / Kruskal MST | Kruskal MST | `MazeAlgorithm::KruskalMst` (1) |
| 回溯 / DFS | DFS Backtracking | `MazeAlgorithm::DepthFirstSearch` (2) |
| 分支限界 / BFS | BFS Branch-and-Bound | `MazeAlgorithm::BreadthFirstSearch` (3) |

界面贴出的复杂度描述（`mainwindow.cpp:397-400`）：
- 分治：`O(V) 时间 / O(V) 空间`
- Kruskal：`O(E α(V)) 时间 / O(V+E) 空间`
- DFS：`O(V+E) 时间 / O(V) 空间`
- BFS 分支限界：`O(E log E) 时间 / O(V+E) 空间`

## 7. 合法性验证

所有算法生成后，`validatePerfect()` 执行两层校验确保结果确实是完美迷宫：

1. **连通性检查**：从起点 BFS，统计可达格子数。必须等于总格子数 V
2. **无环检查**：统计迷宫的总边数。生成树必须满足 $E = V - 1$

两个条件同时满足时，迷宫才是合法的完美迷宫。

## 8. 遗传算法中的算法混合使用

在后续的遗传算法优化中（`MazeOptimizer`），当 `useMixedAlgorithms = true` 时，初始种群会**轮询使用四种算法**生成初始个体：

```cpp
// maze_optimizer.cpp:101-108
MazeAlgorithm MazeOptimizer::algorithmForIndex(int index) {
    static constexpr MazeAlgorithm kAllAlgorithms[4] = {
        MazeAlgorithm::DivideAndConquer,
        MazeAlgorithm::KruskalMst,
        MazeAlgorithm::DepthFirstSearch,
        MazeAlgorithm::BreadthFirstSearch
    };
    return kAllAlgorithms[((index % 4) + 4) % 4];
}
```

这意味着 GA 种群的第一个个体用分治法生成，第二个用 Kruskal，第三个用 DFS，第四个用 BFS，第五个又回到分治……以此类推。这样做的目的是**增加初始种群多样性**，让遗传算法有更丰富的基因库进行交叉变异。
