# AI 玩家输入 JSON 接口

迷宫设计组导出的文件严格包含以下五个字段：

```json
{
  "maze": [["#", "#"], ["S", "E"]],
  "B": [11, 13, 9, 15],
  "PlayerSkills": [[8, 4], [2, 0], [4, 2], [6, 3]],
  "minRouds": 20,
  "CoinConsumption": 5
}
```

- `maze`：二维字符数组，界面中的矩阵行列数就是导出后的实际尺寸。
- `B`：按固定击败顺序排列的 BOSS 血量。
- `PlayerSkills`：每项为 `[伤害, 冷却回合]`，至少一个技能冷却为 0。
- `minRouds`：允许完成整场 BOSS 战的回合数。字段名沿用课程接口中的拼写。
- `CoinConsumption`：超出限定回合后复活所需消耗的金币。

导出器按课程样例保持字段顺序：`maze`、`B`、`PlayerSkills`、`minRouds`、`CoinConsumption`；矩阵中的每一行写在单独一行，不使用 Qt 默认的多行数组展开格式。

矩阵字符：

- `#`：墙壁
- 空格：普通通路
- `S`：唯一起点
- `B`：唯一 BOSS 场景，独立于终点并位于通向终点的必经路径上
- `E`：唯一终点
- `G`：金币，价值 50
- `T`：陷阱，价值 -30

除 `#` 外均为可通行位置。上下左右相邻的可通行位置之间存在边。
