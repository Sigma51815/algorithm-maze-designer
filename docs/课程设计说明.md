# 迷宫设计任务课程设计说明

## 1. 需求对应

| 任务书要求 | 项目实现 |
|---|---|
| 分治、MST、回溯/DFS、分支限界/BFS 生成迷宫 | 界面算法下拉框提供四种方法，并动态播放开墙过程 |
| 无孤立区域且存在唯一通路 | 每次生成后执行连通性检查，并验证 `E=V-1` |
| 尺寸可容纳较多资源 | 支持 `5 x 5` 到 `31 x 31`，默认 `15 x 15` |
| 输出起点、终点、墙、通路、金币、陷阱和 BOSS | 画布显示全部信息，JSON 提供扩展矩阵和结构化字段 |
| 动态规划输出最多资源与最优路径 | 显示最大资源、步数、首次访问格数并播放蓝色路径 |
| 分支限界输出最少回合与最优技能序列 | 支持多 BOSS、多技能、冷却约束并验证结果 |
| BOSS 战过程可视化 | 独立战斗窗口展示血条、伤害、技能冷却、BOSS 切换和最优序列自动播放 |
| 输出限定回合数和复活金币 | 由最少回合加可调余量得到限定回合，复活金币可设置 |

## 2. 四种迷宫生成算法

迷宫把每个格子视为顶点、相邻格子的通路视为边。四种方法最终都构造一棵覆盖全部格子的生成树，因此天然满足连通和唯一通路。

| 算法 | 核心方法 | 时间复杂度 | 空间复杂度 | 典型风格 |
|---|---|---:|---:|---|
| 分治 | 递归拆成两个子区域，各自成树后只增加一条跨区通路 | `O(V)` | 均衡时 `O(log V)` 递归栈 | 分区结构明显 |
| Kruskal MST | 随机边排序，并查集只接受连接不同集合的边 | `O(E log E)` | `O(V+E)` | 分支较均衡 |
| 回溯 DFS | 随机深入未访问邻格，死路时退栈 | `O(V+E)` | `O(V)` | 长走廊较多 |
| BFS 分支限界 | 用优先队列按深度下界扩展候选边，随机代价打散同层分支，访问标记剪去重复和成环状态 | `O(E log E)` | `O(E)` | 分支随机且避免梳齿状退化 |

合法性检查先从起点 BFS 统计可达顶点数，再统计无向边数。满足“全部可达”且 `E=V-1` 时，图是一棵树，即完美迷宫。

## 3. 动态规划资源收集

完美迷宫是一棵树，起点 `S` 到终点 `E` 的主路径唯一。主路径上的格子必须经过；连接在主路径上的支路可以选择进入并原路返回。

对一棵不包含主路径的支路，以连接点方向为父节点，定义：

```text
f(u) = value(u) + sum(max(0, f(v)))
```

其中 `value(u)` 为金币 `50`、陷阱 `-30` 或普通格 `0`。当子树收益 `f(v)>0` 时进入，否则放弃。最终答案为主路径资源之和加所有正收益支路。根据被选支路递归生成“进入、遍历、返回”的实际路径，所以输出的每一步均为上下左右相邻且不穿墙。

- 时间复杂度：`O(V)`。
- 空间复杂度：`O(V)`。
- 同一资源格可能被路径多次经过，但只在首次经过时计分。

## 4. 分支限界 BOSS 战

搜索状态由“各 BOSS 剩余血量”和“各技能剩余冷却”组成，每层代表一个回合。

- 初始上界：用当前可用的最高伤害技能得到一条可行技能序列。
- 乐观下界：`ceil(剩余总血量 / 最大技能伤害)`，忽略冷却，因此不会高估所需回合。
- 剪枝：当前回合数加下界不可能优于上界时停止扩展。
- 状态记忆：同一状态若已用更少或相同回合到达，则不再重复搜索。
- 分支顺序：优先尝试高伤害技能，以更快收紧上界。

求解结束后，程序再次逐回合仿真技能序列，检查技能冷却、BOSS 顺序和最终血量，避免只报告未经验证的回合数。

## 5. 导出格式

JSON 中 `expandedMatrix` 使用以下字符：

- `#`：墙壁
- `.`：通路或普通格
- `S`：起点
- `E`：终点和 BOSS 所在格
- `G`：金币
- `T`：陷阱

另外提供 `passages`、`resources`、`dynamicProgrammingPlan` 和 `bossBattle` 字段，便于测试程序或助教复核。

## 6. 源码结构

- `src/include/`：项目头文件集合，声明算法模型与图形界面类。
- `src/source/maze.cpp`：四种迷宫算法、合法性校验、资源布置、树形 DP、JSON 输出。
- `src/source/bosssolver.cpp`：BOSS 战分支限界和结果仿真验证。
- `src/source/battlewindow.cpp`：回合制 BOSS 战场景、技能特效和自动播放控制。
- `src/source/mazewidget.cpp`：迷宫、资源、生成过程和路径动画绘制。
- `src/source/mainwindow.cpp`：图形界面、输入参数、结果展示和导出。
- `src/source/main.cpp`：程序入口和自动测试。
