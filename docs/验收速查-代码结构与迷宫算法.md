# 验收速查：代码结构与四种迷宫生成算法

> 面向现场验收。验收流程通常是：分别选择分治法、回溯法、分支限界法、最小生成树算法生成迷宫，再把生成结果复制到测试系统验证。本文件说明每个 `.cpp` 的职责、四种算法的位置、复杂度和生成特性。

---

## 一、现场操作入口

1. 打开程序后，在左侧“① 多方法生成完美迷宫”的“算法”下拉框选择算法。
2. 四个选项与代码枚举一一对应：

| UI 显示 | 枚举值 | 入口函数 | 具体实现 |
|---|---|---|---|
| 分治 | `MazeAlgorithm::DivideAndConquer` | `MazeModel::generate()` | `MazeModel::generateDivideAndConquer()` |
| 贪心 / Kruskal MST | `MazeAlgorithm::KruskalMst` | `MazeModel::generate()` | `MazeModel::generateKruskal()` |
| 回溯 / DFS | `MazeAlgorithm::DepthFirstSearch` | `MazeModel::generate()` | `MazeModel::generateDepthFirst()` |
| 分支限界 / BFS | `MazeAlgorithm::BreadthFirstSearch` | `MazeModel::generate()` | `MazeModel::generateBreadthFirst()` |

核心文件：

- `src/include/core/maze.h`：定义 `MazeAlgorithm` 枚举和 `MazeModel` 接口。
- `src/source/core/maze.cpp`：实现四种迷宫生成算法、资源放置、完美迷宫校验、JSON 导出等核心逻辑。
- `src/source/ui/mainwindow.cpp`：创建算法下拉框，并在 `MainWindow::generateMaze()` 中调用 `maze_.generate(...)`。

## 三、四种生成算法的位置和讲解

### 1. 分治法：`generateDivideAndConquer()`

位置：`src/source/core/maze.cpp`

核心思想：

1. 对当前矩形区域按较长方向随机切成两块。
2. 分别递归生成两个子迷宫。
3. 在分割线处随机开一个洞，把两个子迷宫连接起来。
4. 每次只连接两个已经连通的子区域一次，因此不会产生环；最终全局连通且边数为 `V-1`。

复杂度：

- 时间复杂度：平均 `O(V)`，其中 `V = rows × columns`。
- 空间复杂度：递归栈平均约 `O(log V)`；随机极端切分时最坏可到 `O(V)`。

生成特性：

- 区域分割感明显，常出现较长墙段和少量开口。
- 结构比较规整，适合现场展示“递归划分再合并”的思想。
- 迷宫整体连通，但路径风格不像 DFS 那样强烈蜿蜒。

### 2. 最小生成树算法：`generateKruskal()`

位置：`src/source/core/maze.cpp`

核心思想：

1. 把每两个相邻格子的墙看作候选边。
2. 随机打乱所有候选边。
3. 用并查集判断两个格子是否已经连通。
4. 只有当候选边连接两个不同连通块时才凿开墙。
5. 最终选出 `V-1` 条边，得到一棵随机生成树，也就是完美迷宫。

复杂度：

- 候选边数量 `E ≈ 2V`，网格图中 `E = O(V)`。
- 洗牌 `O(E)`，并查集合并/查询近似 `O(E α(V))`。
- 总体可视为 `O(V)`；空间复杂度 `O(V + E)`，网格中也是 `O(V)`。

生成特性：

- 随机性强，整体比较均匀。
- 分支分布较自然，不明显偏向某个方向。
- 适合解释“用最小生成树保证连通且无环”。

### 3. 回溯法 / DFS：`generateDepthFirst()`

位置：`src/source/core/maze.cpp`

核心思想：

1. 从起点开始，把当前格子压入栈。
2. 随机选择一个未访问邻居，凿墙前进。
3. 如果当前格子没有未访问邻居，就从栈中回退。
4. 重复直到所有格子都访问过。

复杂度：

- 时间复杂度：`O(V + E)`，网格中 `E = O(V)`，因此约为 `O(V)`。
- 空间复杂度：访问标记和栈都是 `O(V)`。

生成特性：

- 容易产生长走廊和深死胡同。
- 视觉上蜿蜒感强，适合现场展示“走到尽头再回溯”。
- 路径往往较长，AI 容易在分支里绕路。

### 4. 分支限界法 / BFS 边界扩展：`generateBreadthFirst()`

位置：`src/source/core/maze.cpp`

核心思想：

1. 从随机根节点开始，把可扩展边放入优先队列。
2. 每条候选边都有一个下界 `lowerBound`，由深度、来源格子的分叉度和随机扰动组成。
3. 每次取下界最小的候选边扩展，相当于优先扩展当前更有利的分支。
4. 如果来源格子的度数发生变化，就重新计算下界再入队，保证判断基于最新状态。
5. 外层最多尝试 12 次，保留主路径更长、死胡同更多的一次。

复杂度：

- 单次尝试中，候选边入队/出队主要受 `E` 控制，优先队列操作为 `O(log E)`。
- 单次约 `O(E log E)`；尝试 12 次是常数倍，总体仍可写作 `O(V log V)`。
- 空间复杂度：优先队列、访问数组、度数数组约 `O(V)`。

生成特性：

- 比普通 BFS 更偏“分支限界”：不是简单先进先出，而是根据下界选择扩展边。
- 通过多次尝试保留更长主路径、更有挑战性的迷宫。
- 相比 DFS，结构更受评分规则控制；相比 Kruskal，目标性更强。

## 四、四种算法对比表

| 算法 | 代码函数 | 时间复杂度 | 空间复杂度 | 主要特性 | 适合现场强调 |
|---|---|---:|---:|---|---|
| 分治法 | `generateDivideAndConquer()` | 平均 `O(V)` | 平均 `O(log V)` | 区域感强、墙段长、开口少 | 递归分割 + 子问题合并 |
| 最小生成树 | `generateKruskal()` | 约 `O(V)` | `O(V)` | 随机均匀、用并查集避免成环 | `V-1` 条边保证完美迷宫 |
| 回溯法 | `generateDepthFirst()` | `O(V)` | `O(V)` | 长走廊、深死胡同、蜿蜒明显 | 栈 + 走不动就回退 |
| 分支限界法 | `generateBreadthFirst()` | `O(V log V)` | `O(V)` | 优先队列选边，多次尝试保留更难结构 | 下界估计 + 优先扩展 |

## 五、复制到测试系统前的检查点

生成迷宫后，程序内部会用 `MazeModel::validatePerfect()` 验证：

- 所有格子连通。
- 无环。
- 边数满足 `E = V - 1`。

这三个条件说明生成结果是“完美迷宫”。导出给测试系统时，主要走 `MazeModel::toCrossTestJson()` 和 `MazeSaver` 相关逻辑，确保字段格式与测试系统约定一致。
