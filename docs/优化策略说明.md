# 迷宫难度优化策略

> 本文档描述当前项目中用于**提升迷宫区分度与难度适中性**的优化策略，包括遗传算法（GA）、D/B/C 适应度函数设计、资源分布策略。RL 玩家与协同进化模块已经删除。

---

## 一、优化总览

```
基础迷宫 ──→ GA 优化 ──→ 高区分度迷宫
                │
                ├─ 适应度 = D/B/C 加权公式
                ├─ 对抗资源分布（陷阱放非主路径分叉口，金币藏深分支）
                └─ 种群进化（交叉/变异/选择）
```

## 二、GA 遗传算法优化

### 2.1 流程

```
runOptimizer() → MazeOptimizer::run() on QThread (8MB 栈)
  │
  ├─ 初始种群: randomChromosome × popSize
  │    ├─ 从选中的基础算法生成迷宫拓扑
  │    └─ 按模式放置资源（对抗/智能/随机）
  │
  └─ 迭代 (generations 代):
       │
       ├─ [精英保留] best → nextGen[0] 直接保留
       ├─ [选择] 锦标赛选择 tournamentSize=3
       │    └─ 从种群随机取3个，选适应度最高的
       ├─ [交叉] union-find 合并两父代边集
       │    └─ 构建子代生成树 + 补缺边保证 V-1
       ├─ [变异] 30%概率
       │    ├─ 30%: 完全重新生成迷宫
       │    └─ 70%: 边交换（加一条边→BFS找环→删一条环边）
       ├─ [评估] MazeEvaluator::evaluate() 计算 finalFitness
       └─ [进度] emit generationFinished(stats)
              └─ GUI 更新进度条

  └─ 结束: emit finished(best.maze)
       └─ GUI 自动显示优化前 vs 优化后对比表
```

### 2.2 参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 种群大小 | 16 | 4-100 | 每代个体数 |
| 代数 | 30 | 1-200 | 进化迭代次数 |
| 变异率 | 15% | 1-50% | 每个子代变异概率 |
| 锦标赛大小 | 3 | 固定 | 选择压力 |
| 精英保留 | 1 | 固定 | 每代保留最优个体 |

### 2.3 交叉与变异

```
crossover(mazeA, mazeB):
  1. 收集两父代所有边
  2. Union-Find 优先保留不成环的父代边
  3. 扫描网格相邻单元格补缺
  4. 保证子代 = V-1 条边 + 全连通 = 完美迷宫
  5. 重新放置资源

mutate(chrom):
  if rand() < 0.30:
    → 完全重生成
  else:
    → 边交换（加一条边形成环，再删一条环边）
```

## 三、适应度函数

### 3.1 公式

当前 GA 默认使用增强适应度，目标是让不同 3×3 贪心 AI 的得分拉开差距，同时保持迷宫可完成、难度不过易也不过难。

```cpp
// 1. 归一化每个 AI 得分到 [0,1]
idealScore = dpScore / shortestPathLen
aiScore[i] = max(0, remainingResource[i]) / totalSteps[i]
normalizedScore[i] = clamp01(aiScore[i] / idealScore)
未到终点 → 0

// 2. D 区分度：标准差维度 + 极差维度的调和平均
dStddev = clamp01(stddev(normalizedScores) / 0.22)
dRange  = clamp01((max - min) / 0.55)
D = 2 * dStddev * dRange / (dStddev + dRange)

// 3. C 可完成性：到达终点的 AI 比例
C = reachedCount / 4

// 4. B 难度适中：平均得分靠近 0.45
B = clamp01(1 - abs(mean(normalizedScores) - 0.45) / 0.30)

// 5. 加权适应度
finalFitness = 100 * D^0.55 * B^0.30 * C^0.15
```

### 3.2 各项含义

| 指标 | 范围 | 含义 | 优化方向 |
|------|------|------|---------|
| **D / designDiscrimination** | [0, 1] | 不同 AI 归一化得分的离散程度 | ↑ 越高越好 |
| **B / designBalance** | [0, 1] | 平均 AI 得分是否处于适中难度区间 | ↑ 越高越好 |
| **C / designStability** | [0, 1] | 能到达终点的 AI 比例 | ↑ 越高越好 |
| **meanAIScoreRatio** | [0, 1] | 归一化 AI 平均得分，诊断用 | 接近 0.45 |
| **aiScoreSpread** | [0, 1] | 归一化得分极差，诊断用 | ↑ 越高越能区分策略 |
| **finalFitness** | [0, 100] | D/B/C 综合适应度 | GA 按此最大化选择 |

### 3.3 贪心 AI 评估（4 种策略）

```
对每种策略 (ValuePerStep / CautiousCollector / AvoidTraps / EndGoalFirst):
  GreedyPlayer::play(maze, strategy)
    → PlayResult { reachedEnd, remainingResource, totalSteps, collectedCoins, triggeredTraps }

归一化得分:
  reachedEnd=false → 0
  reachedEnd=true  → clamp01((max(0, remainingResource) / totalSteps) / idealScore)
```

### 3.4 topoDifficulty 诊断项

`topoDifficulty` 仍会计算并展示，但当前主 GA 适应度以 `finalFitness` 为准，不再由旧的 regret × 拓扑难度公式主导。

## 四、资源分布策略

### 4.1 三种模式

| 模式 | 顺序 | 金币位置 | 陷阱位置 | 用途 |
|------|------|---------|---------|------|
| **Random** | - | 随机（优先分支） | 1/3 主路径 | 基础生成 |
| **Smart** | 金币先 | 50%死胡同→深分支→随机 | 非主路分叉口 | GA 智能模式 |
| **Adversarial** | 陷阱先 | 深分支(≥3)→中深→随机（排除死胡同） | 非主路径分叉口 70% | GA 对抗模式（默认 ON） |

### 4.2 对抗模式为什么更适合区分 AI

```
Smart (帮助AI):               Adversarial (对抗AI):
  [金币] 放死胡同              [陷阱] 先放，抢占非主路径分叉口
         → AI 探索死胡同                 → 探索型 AI 更容易踩陷阱
           必捡金币          [金币] 后放，藏在分支深处(branchDepth≥3)
  [陷阱] 放非主路分叉口              → AI 需要深入探索才能找到
         → AI 走主路安全               → 且死胡同不给金币（浪费探索）
```

## 五、RL / 协同进化清理

旧版本曾包含 `rl_player` 与 `coevolution`，但当前代码已经删除这两组模块：

| 删除项 | 当前状态 |
|------|------|
| `src/include/ai/rl_player.h` / `src/source/ai/rl_player.cpp` | 已删除 |
| `src/include/optimization/coevolution.h` / `src/source/optimization/coevolution.cpp` | 已删除 |
| `EvaluatorConfig::evaluateAgainstRL`、`rlScore`、`regretRL` | 已从评估结构中移除 |
| `saveQTable` / `loadQTable` / `saveCoEvolResult` 的 maze_saver 接口 | 已移除 |

删除原因：协同进化使用的 `dpScore - rlScore` 是单对手遗憾值目标，与当前“多策略区分度”目标不一致；RL 自测也有随机性，容易给主流程带来不稳定噪声。

保留项：`qlearning_optimizer` 仍存在，它是独立的 Q-Learning 边操作精调器，不依赖已删除的 RL 玩家，也不参与主 GA 适应度评估。

## 六、GUI 操作路径

```
1. 基础生成:
   选算法 → 设尺寸 → [可选: 勾选 GA] → 点「生成」
   → 迷宫渲染 → 验证标签

2. GA 优化:
   勾选「启用遗传算法优化」→ 设种群/代数/变异率
   → 勾选「对抗模式」(默认 ON)
   → 点「生成」
   → 等待迭代 → 自动显示对比表

3. 查看结果:
   对比表: D 区分度 / B 难度适中 / C 可完成性 / AI 得分均值比 / AI 得分极差 / finalFitness
   结论: ✓ 优化有效(适应度提升) 或 ✗ 优化未提升

4. 保存:
   点「保存最佳迷宫」→ JSON 文件（包含迷宫数据 + GA 配置 + 适应度）
```

## 七、已知局限与改进方向

| 问题 | 说明 | 改进方向 |
|------|------|---------|
| D 依赖 4 种内置策略 | 真实参赛 AI 可能有不同启发式 | 增加更多局部策略做交叉评估 |
| B 的理想均值固定为 0.45 | 不同尺寸/资源密度下理想难度可能不同 | 根据尺寸和资源总量做自适应参数 |
| 未到终点记 0 分 | 能防止全员失败误判，但会压低极难图的平均分 | 额外记录失败类型用于诊断 |
| `topoDifficulty` 仍作为诊断项 | 当前主 GA 使用 finalFitness，拓扑难度不直接主导选择 | 保留用于 UI 分析和后续调参 |
| 历史 RL 自测随机性 | 已删除模块的历史问题，当前主流程不受影响 | 保持 RL 玩家与协同进化从主流程移除 |
