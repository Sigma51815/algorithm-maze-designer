# 迷宫难度优化策略

> 本文档详细描述项目中用于**提升迷宫难度**的所有优化策略，包括遗传算法（GA）、协同进化（GA+RL）、适应度函数设计、资源分布策略。

---

## 一、优化总览

```
基础迷宫 ──→ GA 优化 ──→ 高难度迷宫
                │
                ├─ 适应度 = regret × 拓扑难度因子
                ├─ 对抗资源分布（陷阱抢分叉口，金币藏深分支）
                └─ 种群进化（交叉/变异/选择）
                
协同进化：
  GA 迷宫种群 ←──交替进化──→ RL AI 种群
  迷宫变难                    AI 变强
```

## 二、GA 遗传算法优化

### 2.1 流程

```
runOptimizer() → MazeOptimizer::run() on QThread (8MB 栈)
  │
  ├─ 初始种群: randomChromosome × popSize
  │    ├─ 从选中的基础算法生成迷宫拓扑
  │    └─ 按模式放置资源（对抗/智能/随机）
  │
  └─ 迭代 (generations 代):
       │
       ├─ [精英保留] best → nextGen[0] 直接保留
       │
       ├─ [选择] 锦标赛选择 tournamentSize=3
       │    └─ 从种群随机取3个，选适应度最高的
       │
       ├─ [交叉] union-find 合并两父代边集
       │    └─ 构建子代生成树 + 补缺边保证 V-1
       │
       ├─ [变异] 30%概率
       │    ├─ 30%: 完全重新生成迷宫
       │    └─ 70%: 边交换（加一条边→BFS找环→删一条环边）
       │
       ├─ [评估] evaluateFitness()
       │    └─ MazeEvaluator::evaluate() 计算适应度
       │
       └─ [进度] emit generationFinished(stats)
              └─ GUI 更新进度条

  └─ 结束: emit finished(best.maze)
       └─ GUI 自动显示优化前 vs 优化后对比表
```

### 2.2 参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 种群大小 | 16 | 4-100 | 每代个体数 |
| 代数 | 30 | 1-200 | 进化迭代次数 |
| 变异率 | 15% | 1-50% | 每个子代变异概率 |
| 锦标赛大小 | 3 | 固定 | 选择压力 |
| 精英保留 | 1 | 固定 | 每代保留最优个体 |

### 2.3 交叉算法

```
crossover(mazeA, mazeB):
  1. 收集两父代所有边 → targetEdges (最多 V-1 条)
  2. Union-Find 遍历 targetEdges:
     - 若边的两端在不同集合 → 加入子代，合并集合
  3. 补缺: 扫描网格所有相邻单元格对
     - 若未连接且不在同集合 → 加入子代
  4. 保证: 子代 = V-1 条边 + 全连通 = 完美迷宫
  5. 重新放置资源（使用当前配置的对抗/智能模式）
```

### 2.4 变异算法

```
mutate(chrom):
  if rand() < 0.30:
    → 完全重生成: randomChromosome() 全新迷宫
  else:
    → 边交换 (edgeSwap):
      1. 收集所有"墙"：相邻但未连接的单元格对
      2. 随机选一堵墙 → 凿开（添加边）
      3. BFS 从新边的任一端出发找另一端的路径 → 找到环
      4. 从环中随机删一条边（不含刚加的） → 恢复生成树
      5. 重新放置资源
```

## 三、适应度函数

### 3.1 公式

```
finalFitness = coinMissRate × 50
             + trapHitRate  × 30
             + pathInefficiency × 20
             + topoWeight × topoDifficulty × 50
```

### 3.2 各项含义

| 指标 | 范围 | 含义 | 优化方向 |
|------|------|------|---------|
| **coinMissRate** | [0, 1] | AI 漏捡的金币比例 | ↑ 越高越好（AI 找不到金币） |
| **trapHitRate** | [0, 1] | AI 踩中陷阱的比例 | ↑ 越高越好（AI 狂踩陷阱） |
| **pathInefficiency** | [0, ∞) | AI 路径 / 最短路径 - 1 | ↑ 越高越好（AI 绕远路） |
| **topoDifficulty** | [0, 5] | 拓扑复杂度因子 | ↑ 越高越好（迷宫结构复杂） |
| **finalFitness** | — | 综合难度分 | GA 按此最大化选择 |

### 3.3 贪心 AI 评估（4 种策略取最差）

```
对每种策略 (ValuePerStep / NearestFirst / AvoidTraps / EndGoalFirst):
  GreedyPlayer::play(maze, strategy)
    → PlayResult { remainingResource, totalSteps, collectedCoins, triggeredTraps }
    
取 最差分:
  coinMissRate      = max(1 - collectedCoins/总金币数)  ← 哪个策略漏得最多
  trapHitRate       = max(triggeredTraps/总陷阱数)       ← 哪个策略踩得最多
  pathInefficiency  = max((totalSteps - 最短路径) / 最短路径)
```

### 3.4 topoDifficulty 计算

```
topoDifficulty = clamp( sum , 0 , 5 ):

  死胡同密度    deadEnds / cellCount × 2.0
  最长走廊      min(1, longestCorridor / 10) × 0.5
  路径迂回度    (solutionLength / shortestPath - 1) × 0.5
  分叉口密度    junctions / cellCount × 1.0
```

| 因子 | 物理含义 | 对难度的影响 |
|------|---------|------------|
| 死胡同多 | 探索容易进死路 | AI 反复回头浪费时间 |
| 走廊长 | 长直通道 | AI 只能直线前进，减少决策机会 |
| 路径迂回 | 最优路径本身很长 | 即使最优也需走很多步 |
| 分叉口多 | 决策点多 | AI 容易选错方向 |

## 四、资源分布策略

### 4.1 三种模式

| 模式 | 顺序 | 金币位置 | 陷阱位置 | 用途 |
|------|------|---------|---------|------|
| **Random** | — | 随机（优先分支） | 1/3 主路径 | 基础生成 |
| **Smart** | 金币先 | 50%死胡同→深分支→随机 | 非主路分叉口 | GA 智能模式 |
| **Adversarial** | 陷阱先 | 深分支(≥3)→中深→随机（排除死胡同） | **全部**分叉口 70% | GA 对抗模式（默认 ON） |

### 4.2 对抗模式为什么更难

```
Smart (帮助AI):               Adversarial (对抗AI):
  [金币] 放死胡同              [陷阱] 先放，抢占所有分叉口
         → AI 探索死胡同                 → AI 走到岔路口就踩陷阱
           必捡金币          [金币] 后放，藏在分支深处(branchDepth≥3)
  [陷阱] 放非主路分叉口              → AI 需要深入探索才能找到
         → AI 走主路安全               → 且死胡同不给金币（浪费探索）
```

## 五、协同进化（GA + RL）

### 5.1 核心思想

单独用 GA 优化只能对付**固定的贪心 AI**。如果 AI 也会学习变强，迷宫需要跟着进化——这就是协同进化。

### 5.2 流程

```
CoEvolution::run():

初始化:
  ├─ GA 迷宫种群: popSize 个迷宫（4 算法混合）
  └─ RL 玩家: Q 表全零

循环 cycles 轮:
  │
  ├─ [GA 阶段] gaGenerations 代:
  │    ├─ 评估: evalRLFitness = dpVal - rl.totalResource
  │    │         (迷宫 DP 最优 vs RL 实际得分，差距越大越难)
  │    ├─ 选择/交叉/变异（同 GA 流程）
  │    └─ 精英保留
  │
  ├─ [RL 阶段] 训练:
  │    ├─ 选出 GA 种群中 top-K 个最难迷宫
  │    ├─ rlPlayer.trainOnMazes(hardestMazes)
  │    │    └─ Q-Learning 学怎么在这些迷宫中获得更高分数
  │    └─ RL 变强后，下一轮 GA 需要对抗更强的 AI
  │
  └─ 记录: 最佳适应度、RL 分数、拓扑难度

输出: 最佳迷宫 + 统计数据
```

### 5.3 与普通 GA 的区别

| 方面 | GA | 协同进化 |
|------|-----|---------|
| 评估对象 | 4 种固定贪心策略 | 会进化的 RL 玩家 |
| 适应度 | coinMissRate×50 + ... | dpVal - rlScore |
| AI 会变强？ | 不会 | 会，每轮 Q-Learning 训练 |
| 迷宫需要 | 只需强于贪心 | 需要持续强于进化的 RL |
| 用途 | 快速产生高难度迷宫 | 研究性对抗 |

## 六、GUI 操作路径

```
1. 基础生成:
   选算法 → 设尺寸 → [可选: 勾选 GA] → 点「生成」
   → 迷宫渲染 → 验证标签

2. GA 优化:
   勾选「启用遗传算法优化」→ 设种群/代数/变异率
   → 勾选「对抗模式」(默认 ON)
   → 点「生成」
   → 等待迭代 → 自动显示对比表

3. 查看结果:
   对比表: 金币漏拾率 / 陷阱命中率 / 路径迂回度 / 综合难度分
   结论: ✓ 优化有效(难度分提升) 或 ✗ 优化未变难

4. 保存:
   点「保存最佳迷宫」→ JSON 文件（包含迷宫数据 + GA 配置 + 适应度）
```

## 七、已知局限与改进方向

### 7.1 GA 适应度公式局限

| 问题 | 说明 | 改进方向 |
|------|------|---------|
| pathInefficiency 无上界 | AI 走满 maxSteps 时可达 ~2000 | clamp 到 [0, 5] |
| topoDifficulty 饱和 | solutionLength/shortestPath≥9 即饱和 | 降低 weight 或调整因子 |
| 跨策略取最大 | 3 个指标可能来自不同策略 | 对每个策略独立算 fitness |

### 7.2 RL 训练局限

| 问题 | 说明 |
|------|------|
| ε 衰减过慢 | 2000 episodes 后 ε 仍 ~0.18，远未到 0.03 |
| 每迷宫重置 ε | trainOnMazes 对每个迷宫重置 ε=0.5 |
| 距离奖励振荡 | 走近+0.01、走远不惩罚 → agent 可靠振荡蹭奖 |

### 7.3 协同进化局限

| 问题 | 说明 |
|------|------|
| bestFitness 初始 0 | 负适应度永不记录 |
| 首轮 Q 表全零 | 第一次评估是随机噪声 |
| RL 只用 dpVal-rlScore | 与主 GA 的复合公式不一致 |
