# 迷宫设计组验收对照清单

## 阶段测试：多方法迷宫生成（12 分）

| 评分点 | 当前实现 | 验收操作 |
|---|---|---|
| 四种算法 | 分治、Kruskal MST、回溯 DFS、严格分层 BFS | 下拉框依次生成四次 |
| 完美迷宫 | 自动检查连通性和 `E=V-1` | 查看绿色校验文字 |
| 生成过程可视化 | 逐边开墙动画 | 点击“生成并播放过程” |
| 复杂度与风格 | 界面说明 + 课程设计说明表格 | 切换算法并讲解指标差异 |
| 挑战性 | 树直径作为 S-E 主线，显示解路径、死路、分叉点、最长走廊 | 使用相同尺寸比较四种算法 |

## 阶段测试：动态规划资源收集（9 分）

| 评分点 | 当前实现 | 验收操作 |
|---|---|---|
| 最大资源正确 | 树形 DP；小迷宫自动测试用穷举连通子图独立核对 | 运行 `--self-test` |
| 路径合法 | 每步检查相邻通路，资源仅首次计分 | 点击“DP 求最优路径” |
| 可视化 | 蓝色路径逐步播放 | 观察迷宫画布 |
| 具有权衡 | 主线布置部分陷阱，金币优先放在支路 | 修改资源数后重新求解 |

## 阶段测试：分支限界 BOSS 战（9 分）

| 评分点 | 当前实现 | 验收操作 |
|---|---|---|
| 最少回合正确 | 分支限界求解；小用例用无启发式 BFS 独立核对 | 运行 `--self-test` |
| 技能序列可验证 | 逐回合仿真血量和冷却 | 查看求解结果 |
| 上下界与剪枝 | 贪心序列作上界；冷却感知伤害容量作乐观下界；状态记忆剪枝 | 展示搜索/剪枝状态数 |
| 战斗可视化 | 独立战斗窗口、血条、伤害、冷却、BOSS 切换、最优序列自动播放 | 点击“打开可视化战斗动画” |

## 仍需按课程现场材料确认

- 若老师另发指定 API、输入输出格式或平台必做题，需要按最终接口增加适配层。
- 交叉测试前应从多个随机种子中选择挑战性指标较好的迷宫，并导出 JSON。
- 课程报告须使用《计算机学报》模板且不少于 7 页。
- 报告必须包含四个算法的伪代码或流程图、复杂度分析过程、实验图表、创新点和真实分工。
- 分工与贡献比例需要全员确认，否则评分手册规定可能统一按贡献系数 1.00 处理。
