# 四人协作约定

本仓库采用 `main` + 功能分支的协作方式。

## 分支命名

- `feature/name`：新增功能
- `fix/name`：修复问题
- `docs/name`：文档修改
- `test/name`：测试补充

## 提交流程

1. 从最新 `main` 创建自己的功能分支。
2. 每次提交只处理一个明确问题，并写清提交信息。
3. 推送分支后创建 Pull Request。
4. 至少由另一名组员检查代码后再合并。
5. 合并前必须保证项目可以编译，`maze_designer.exe --self-test` 全部通过。

## 建议分工

| 角色 | 主要内容 |
|---|---|
| 成员 1 | 分治、Kruskal MST 与迷宫合法性测试 |
| 成员 2 | DFS、BFS 迷宫生成与动画展示 |
| 成员 3 | 动态规划资源收集与路径可视化 |
| 成员 4 | BOSS 分支限界、测试、报告整合 |

实际分工应在课程报告中按每位成员的真实贡献填写。
